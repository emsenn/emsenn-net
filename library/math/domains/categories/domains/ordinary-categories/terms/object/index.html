2025-12-26 One of the two kinds of data in a category — the things that morphisms go between.

An object is one of the two kinds of data in a category — the things that morphisms go between.

What counts as an object depends on the category. In Set, objects are sets. In Top, objects are topological spaces. In a group G viewed as a one-object category, there is exactly one object (the group itself), and morphisms are the group elements. In a partial order viewed as a thin category, each element of the poset is an object.

Objects in a category are not studied in isolation — they are studied through their morphisms. Two objects that are isomorphic (connected by an invertible morphism) are interchangeable for all categorical purposes. The categorical perspective holds that an object is characterized by its relationships to other objects, not by any internal constitution.

This principle is made precise by the Yoneda lemma: an object A is completely determined (up to isomorphism) by the functor Hom(−, A) that records all morphisms into A. Two objects are isomorphic if and only if they have the same morphisms into them from every other object. An object has no hidden internal state that the morphisms fail to capture — if two objects look the same from the outside, they are the same.
