<p>Learning goal: understand how decision support and synchronization artifacts turn uncertainty into executable coordination.</p> <h2 id="decision-support-turn-uncertainty-into-explicit-triggers">Decision support: turn uncertainty into explicit triggers</h2> <p>The purpose of decision support is to prevent last-minute decisions made without the right information <a href="#ref-usarmy2019adp5_0" class="citation">[usarmy2019adp5_0]</a>. You do this by pre-identifying:</p> <ul> <li>decision points,</li> <li>triggers (what observable change means you must decide),</li> <li>information requirements,</li> <li>and pre-planned actions.</li> </ul> <p>A disciplined way to express information requirements is through <a href="../terms/ccir.md" class="link-internal">CCIR</a>: a small set of <a href="../terms/pir.md" class="link-internal">PIR</a> and <a href="../terms/ffir.md" class="link-internal">FFIR</a> tightly tied to decision points.</p> <p>If a question is not tied to a decision, it is probably not a CCIR.</p> <h2 id="synchronization-make-distributed-actions-converge">Synchronization: make distributed actions converge</h2> <p>Synchronization is not “coordination in general.” It is aligning time, space, and priorities so multiple units’ actions add up.</p> <p>A synchronization matrix typically aligns:</p> <ul> <li>phases,</li> <li>tasks by unit or function,</li> <li>key events,</li> <li>decision points,</li> <li>and constraints and priorities.</li> </ul> <p>The point is convergence: the minimum set of rules that keeps distributed action coherent.</p> <h2 id="exercise">Exercise</h2> <p>Create a minimal DSM:</p> <ul> <li>5 decision points,</li> <li>each with a trigger,</li> <li>who decides,</li> <li>what action follows.</li> </ul> <p>Then create a minimal synchronization matrix with three phases and three units or functions.</p>
