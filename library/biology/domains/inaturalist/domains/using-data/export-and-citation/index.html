Export and citation

Proper export and citation preserve attribution and clarify data provenance. This matters for reproducibility and ethical reuse.

Practice guidance

  • Use iNaturalist export tools or GBIF downloads for structured data.
  • Cite both iNaturalist and the specific dataset DOI when using GBIF records.
  • Record the export date and filters used.

What goes wrong if you do this poorly

Without clear citation, downstream users cannot evaluate provenance or reproduce analyses, weakening credibility and trust.

Sources

  • iNaturalist data export guidance: https://www.inaturalist.org/pages/help#export
  • GBIF citation guidance: https://www.gbif.org/citation-guidelines

Template citation example

iNaturalist contributors. iNaturalist Research-grade Observations. Accessed YYYY-MM-DD. Dataset DOI: [GBIF DOI]. Filtered for [place/taxon/date].

Versioning and reproducibility

  • Record the export date and query parameters.
  • Save the exact CSV or download identifier used in analysis.
  • Note any post-processing steps applied after export.
