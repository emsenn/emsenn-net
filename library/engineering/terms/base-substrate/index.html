2026-03-20 A Base is a small category B — a collection of objects (positions) and morphisms (transitions) with composition and identities, carrying no Grothendieck topology — that provides the minimal indexing structure over which a presheaf assigns fibers. The defining structure: Mode-1 substrate, providing positional indexing without topological constraint. A Base imposes no covering conditions; every presheaf on B is permissible. In the relational universe, the history poset (T, ≤) is the Base: it provides histories t ∈ T (positions) and the ordering t ≤ t' (transitions) without specifying which families of histories cover another. A Foundation is a Base enriched with a Grothendieck topology.

Base

Formal definition

A Base is a small category B\mathcal{B}:

B=(Obj(B),  Mor(B),  ∘,  id)\mathcal{B} = (\mathrm{Obj}(\mathcal{B}),\; \mathrm{Mor}(\mathcal{B}),\; \circ,\; \mathrm{id})

where:

  • Obj(B)\mathrm{Obj}(\mathcal{B}) is the object set — the positions available for fiber assignment; each object b∈Bb \in \mathcal{B} is a position in the indexing domain; any superstructure defined over B\mathcal{B} assigns something to every position
  • Mor(B)\mathrm{Mor}(\mathcal{B}) is the morphism set — the transitions between positions; for each morphism f:b→b′f : b \to b', the superstructure provides a corresponding map between fibers; the morphisms encode the covariance or contravariance structure of the assignment
  • ∘\circ is composition: for f:b→b′f : b \to b' and g:b′→b′′g : b' \to b'', the composite g∘f:b→b′′g \circ f : b \to b''; the superstructure must respect this composition
  • idb\mathrm{id}_b is the identity morphism at each bb; the superstructure sends identity to identity

A Base carries no additional structure. In particular:

  • No Grothendieck topology: no specification of which families of morphisms “cover” a position; no covering conditions; no notion of compatible local data needing global amalgamation
  • No monoidal structure (unless explicitly added for a different purpose)
  • No enrichment (unless explicitly required by the fiber category)

Four invariants. B\mathcal{B} is a base iff it satisfies:

  1. Positional completeness: every object b∈Bb \in \mathcal{B} is available as a base point for fiber assignment. There are no “masked” positions. Any presheaf or superstructure that nominally lives over B\mathcal{B} assigns a fiber at every bb — a partial assignment is not a superstructure on B\mathcal{B} but on a restriction.

  2. Transitional coherence: composition in B\mathcal{B} is associative and unital. These are the category axioms: (h∘g)∘f=h∘(g∘f)(h \circ g) \circ f = h \circ (g \circ f) and f∘idb=f=idb′∘ff \circ \mathrm{id}_b = f = \mathrm{id}_{b'} \circ f for f:b→b′f : b \to b'. The superstructure’s restriction maps must respect this composition contravariantly: F(g∘f)=F(f)∘F(g)F(g \circ f) = F(f) \circ F(g).

  3. Topological neutrality: B\mathcal{B} carries no Grothendieck topology. No covering sieves are specified; no covering conditions are imposed; every presheaf on B\mathcal{B} is a valid assignment. This is the defining difference between a Base and a Foundation: a Foundation is a Base equipped with a topology JJ that selects which presheaves (those satisfying the sheaf condition) are the coherent ones. The Base makes no such selection — it provides the positions and transitions and nothing more.

  4. Preservation: the base persists unchanged through the operation of any superstructure defined over it. The presheaf assigns fibers to the base’s positions and restriction maps to the base’s morphisms, but does not alter the base. The base is not consumed, not modified, not specialized. It is the stable indexing domain that remains in place while the superstructure varies.

The canonical base in the relational universe

The history poset (T,≤)(T, \leq) is the canonical base in the relational universe. Viewed as a category:

  • Objects: histories t∈Tt \in T — the positions in the indexing domain; each history is a point at which propositions can be evaluated
  • Morphisms: there is a unique morphism t→t′t \to t' iff t≤t′t \leq t' (the poset ordering); since at most one morphism exists between any pair of objects, this is a thin category — a posetal category
  • Composition: transitivity of ≤\leq: if t≤t′t \leq t' and t′≤t′′t' \leq t'', then t≤t′′t \leq t''
  • Identities: reflexivity of ≤\leq: t≤tt \leq t for every tt

The history poset is a base for the presheaf H:Top→HAnuclH : T^{\mathrm{op}} \to \mathbf{HA}_{\mathrm{nucl}} — the assignment of a Heyting algebra with commuting nuclear pair to each history. The presheaf assigns:

t  ⟼  Ht(fiber at history t)t \;\longmapsto\; H_t \quad \text{(fiber at history } t\text{)}

t≤t′  ⟼  ρt∣t′:Ht′→Ht(restriction map)t \leq t' \;\longmapsto\; \rho_{t|t'} : H_{t'} \to H_t \quad \text{(restriction map)}

The history poset as a base (without topology) imposes only that these assignments are functorial. Any family {Ht}t∈T\{H_t\}_{t \in T} with compatible restriction maps is a presheaf on the base. Whether it is a sheaf — whether compatible local families assemble uniquely — is determined by the topology JJ, which belongs to the Foundation (T,J)(T, J), not to the base TT alone.

The base is minimal: it is the smallest structure needed to parameterize a field. Removing any part of the base (an object, a morphism) changes which presheaves exist. Adding any structure (a topology, a monoidal product) produces a richer substrate — a Foundation, a monoidal base, etc. The base is the irreducible datum of positional indexing.

Posetal bases and their presheaves

When B=(T,≤)\mathcal{B} = (T, \leq) is a poset, the category structure is thin: ∣B(b,b′)∣≤1|\mathcal{B}(b, b')| \leq 1 for all b,b′b, b'. This simplifies presheaves over B\mathcal{B}:

A D\mathcal{D}-valued presheaf on a posetal base (T,≤)(T, \leq) is precisely:

  • A family of objects (F(t))t∈T(F(t))_{t \in T} in D\mathcal{D}
  • A family of morphisms (ρt∣t′:F(t′)→F(t))t≤t′(\rho_{t|t'} : F(t') \to F(t))_{t \leq t'} in D\mathcal{D}
  • satisfying: ρt∣t=idF(t)\rho_{t|t} = \mathrm{id}_{F(t)} and ρt∣t′′=ρt∣t′∘ρt′∣t′′\rho_{t|t''} = \rho_{t|t'} \circ \rho_{t'|t''} for t≤t′≤t′′t \leq t' \leq t''

The thin structure eliminates all naturality conditions beyond the basic functoriality above: there are no parallel morphisms to require commutativity of extra squares. This is why posetal bases arise so naturally in ordered time (histories), partial-order semantics, and Kripke frames — they provide the cleanest form of positional indexing.

Base and representable presheaves

For any base B\mathcal{B} and any object b∈Bb \in \mathcal{B}, the representable presheaf y(b)=B(−,b):Bop→Sety(b) = \mathcal{B}(-, b) : \mathcal{B}^{\mathrm{op}} \to \mathbf{Set} assigns to each b′b' the hom-set B(b′,b)\mathcal{B}(b', b). The Yoneda lemma asserts:

Nat(y(b),F)≅F(b)\mathrm{Nat}(y(b), F) \cong F(b)

for any presheaf FF on B\mathcal{B}. The base’s positions are fully characterized by their representables. For the history poset: y(t)(t′)={∗}y(t)(t') = \{*\} if t′≤tt' \leq t, else ∅\emptyset — the representable at history tt is the downward principal ideal ↓t{\downarrow}t.

The density theorem follows: every presheaf on B\mathcal{B} is a colimit of representables. The base’s own structure generates all the presheaves over it, through colimits.

Base vs. adjacent concepts

Concept Relation to Base
Foundation Base enriched with a Grothendieck topology JJ; a Foundation is a site (B,J)(\mathcal{B}, J)
Substrate General concept; a Base is Mode-1 substrate — preserved, positional, topologically neutral
Field A field IS the superstructure over a base; the base is the substrate of a field
Presheaf A presheaf is the minimal superstructure over a base — the assignment of fibers and restriction maps
Category A base is a small category used as an indexing domain; every small category is a potential base

Open questions

  • Whether the smallness requirement (small category) is essential for the base concept, or whether large bases are admissible in certain settings. In practice, the history category TT is small (countable histories), but there is no logical obstacle to a large base if set-theoretic care is taken.
  • Whether a base morphism (functor f:B→B′f : \mathcal{B} \to \mathcal{B}' between two bases) should be part of the base concept, and how base morphisms interact with the presheaves they induce (the pullback functor f∗:[B′op,D]→[Bop,D]f^* : [\mathcal{B}'^{\mathrm{op}}, \mathcal{D}] \to [\mathcal{B}^{\mathrm{op}}, \mathcal{D}]).
  • Whether the initial object (empty category 0\mathbf{0}) and terminal object (single-object single-morphism category 1\mathbf{1}) of the category of small categories are degenerate bases, and what superstructures over them look like — presheaves on 1\mathbf{1} are just objects of D\mathcal{D}; presheaves on 0\mathbf{0} are the terminal presheaf (empty product).
  • Whether the base concept should be distinguished from the index category concept in the theory of (co)limits — whether indexing for limits and indexing for presheaves/fields are formally the same, and whether a base is always an index category.
