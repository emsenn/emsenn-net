Ideas for the Universal Relational Computer

Ideas Index

This directory collects prospective design notes for future slices of the universal relational computer. Each file is intentionally a plan, not code and not a binding specification: it names recognitions, measurements, topology, and runbooks for a domain so that, when the core system is ready, we can lift the idea into real specs and implementations under docs/app/spec/ and docs/app/arch/.

Shared pattern

Nearly every idea should answer the same questions:

  1. What are the core recognitions?
    • What are the “things that matter” in this domain (objects, events, relationships, states)?
  2. How do topology and measurement apply?
    • What graphs/structures relate these recognitions?
    • What observables do we care about over time?
  3. What runbooks live here?
    • What workflows/actions manipulate these recognitions (diagnose, intervene, explore, compare, publish, etc.)?
  4. How does this connect to other domains?
    • Which ideas does it naturally touch (health, ecology, MUD, governance, markets, inner work, etc.)?

Future idea notes should follow this pattern, even if only in a lightweight way.
