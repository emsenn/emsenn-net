1. Introduction

The agential semioverse defines individual agents and their execution semantics: an internal agent A=(Σ,K,Π)A = (\Sigma, K, \Pi) selects interaction operators from its skillset KK according to its policy Π\Pi and applies them to a partial state u∈Uu \in \mathcal{U}. The specification addresses single-agent execution in detail but does not address what happens when multiple agents act on the same state concurrently.

In Claude Code, multi-agent coordination is implemented through the Agent tool, which spawns subagents. Each subagent runs as a separate session with its own context. Claude Code provides three coordination mechanisms:

  • Parallel execution. Multiple Agent tool calls in a single message execute concurrently. Subagents can be typed (e.g., general-purpose, Explore, Plan) or custom-defined in .claude/agents/*.md.
  • Worktree isolation. Setting isolation: "worktree" on an Agent call creates a git worktree — an isolated filesystem copy of the repository. Changes in the worktree do not affect the main working tree until explicitly merged.
  • Background execution. Setting run_in_background: true launches an agent non-blockingly. The parent agent is notified when the background agent completes.

In practice, three patterns recur:

  1. Fan-out across directory subtrees. A coordinator agent spawns multiple Agent calls in parallel, each targeting a different directory subtree. With worktree isolation, each worker operates on its own filesystem copy. Without worktree isolation, workers must target disjoint directories to avoid conflicts.

  2. Pipeline composition. One agent’s output feeds the next. Sequential Agent calls pass information through their return values. For example, an audit agent returns a list of broken references; the parent agent passes that list to a repair agent.

  3. Config-driven batch operations. Multiple agents share a configuration artifact (e.g., a JSON file specifying cross-link targets). Each agent reads the config but only modifies files in its assigned region. Only the coordinator may update the config itself.

Formalizing these patterns requires extending the existing framework with region ownership, non-interference conditions, and composition theorems. This document provides that extension. The region formalism models what worktree isolation achieves operationally: guaranteeing that concurrent agents do not interfere with each other’s work.

We assume familiarity with the base structures: the complete Heyting algebra HH with modal closure jj and trace comonad GG; the definable operator algebra Opdef\mathsf{Op}^{\mathrm{def}}; fragments F(H)\mathcal{F}(H) as finitely generated modal-temporal Heyting subalgebras; partial states U=P(H)×P(Opdef)\mathcal{U} = \mathcal{P}(H) \times \mathcal{P}(\mathsf{Op}^{\mathrm{def}}); the composite closure operator S=Sfus∘Ssyn∘Ssem\mathcal{S} = \mathcal{S}_{\mathrm{fus}} \circ \mathcal{S}_{\mathrm{syn}} \circ \mathcal{S}_{\mathrm{sem}} with least fixed point X⋆X^{\star}; handles Handle\mathsf{Handle} with semantic seeds η:Handle→H\eta : \mathsf{Handle} \to H; handle fragments Fτ,XF_{\tau, X}; footprints FootX(τ)=SX(Fτ,X)\mathrm{Foot}_X(\tau) = \mathcal{S}_X(F_{\tau, X}); and interaction operators Int\mathbf{Int} as monotone endomorphisms of U\mathcal{U} satisfying fragment-preservation, closure-compatibility, provenance extension, and failure monotonicity. We also assume the agential extension: agent profiles P(a)=(rolea,goalsa,policya,Skillsa,Toolsa,Mema)P(a) = (\mathrm{role}_a, \mathrm{goals}_a, \mathrm{policy}_a, \mathrm{Skills}_a, \mathrm{Tools}_a, \mathrm{Mem}_a), internal agents A=(Σ,K,Π)A = (\Sigma, K, \Pi), the skill calculus Skilldef\mathsf{Skill}^{\mathrm{def}}, policy-restricted interpretation [ ⁣[s] ⁣]a[\![s]\!]_a, and the agential closure operator Sagent\mathcal{S}_{\mathrm{agent}} with its least fixed point A⋆A^{\star}.

2. Regions

We formalize the notion of a “region” within the fragment lattice that corresponds to a directory subtree in the ASR implementation.

Definition 2.1 (Region). A region R⊆F(H)R \subseteq \mathcal{F}(H) is a downward-closed subset of the fragment lattice: if F∈RF \in R and F′⊆FF' \subseteq F with F′∈F(H)F' \in \mathcal{F}(H), then F′∈RF' \in R.

Downward closure captures the intuition that if a region owns a fragment, it also owns all subfragments of that fragment. In the ASR implementation, a region RdR_d for directory dd contains all fragments whose handles lie under dd in the filesystem hierarchy.

Definition 2.2 (Region generated by a directory). Given a directory path dd and the set Handled={τ∈Handle:τ lies under d}\mathsf{Handle}_d = \{\tau \in \mathsf{Handle} : \tau \text{ lies under } d\}, define the directory region:

Rd={F∈F(H):every generator of F lies in ⋃τ∈HandledFτ,X}.R_d = \{F \in \mathcal{F}(H) : \text{every generator of } F \text{ lies in } \bigcup_{\tau \in \mathsf{Handle}_d} F_{\tau, X}\}.

Then RdR_d is downward-closed by construction, since subfragments of fragments generated by things under dd are themselves generated by elements from things under dd.

Definition 2.3 (Disjoint regions). Two regions R1,R2R_1, R_2 are disjoint if R1∩R2=∅R_1 \cap R_2 = \emptyset: no fragment belongs to both regions.

Definition 2.4 (Region cover). A region cover of F(H)\mathcal{F}(H) is a finite family {Ri}i=1n\{R_i\}_{i=1}^{n} such that every fragment F∈F(H)F \in \mathcal{F}(H) belongs to at least one RiR_i. A region cover is disjoint if the RiR_i are pairwise disjoint.

Lemma 2.5 (Directory subtrees give disjoint regions). If d1,…,dnd_1, \ldots, d_n are directories with pairwise disjoint filesystem subtrees (no did_i is an ancestor of djd_j for i≠ji \neq j, and their subtrees do not overlap), then Rd1,…,RdnR_{d_1}, \ldots, R_{d_n} are pairwise disjoint regions.

Proof. A handle τ\tau lies under at most one did_i (since the subtrees are disjoint in the filesystem), so Fτ,XF_{\tau, X} belongs to at most one RdiR_{d_i}. Fragments are generated by handle denotations; a fragment whose generators come from disjoint handle sets cannot belong to two distinct directory regions.

3. Region-Restricted Agents

We define what it means for an agent to operate within a region.

Definition 3.1 (Region-restricted skillset). Given an agent A=(Σ,K,Π)A = (\Sigma, K, \Pi) and a region RR, define the region-restricted skillset:

K∣R={E∈K:for all u∈U, if F∉R then E(u)∣F=u∣F}.K|_R = \{E \in K : \text{for all } u \in \mathcal{U}, \text{ if } F \notin R \text{ then } E(u)|_F = u|_F\}.

That is, K∣RK|_R contains exactly those interaction operators in KK that act as the identity on fragments outside RR.

Definition 3.2 (Region-restricted agent). The region-restricted agent A∣R=(Σ,K∣R,Π∣R)A|_R = (\Sigma, K|_R, \Pi|_R) where:

  • Σ\Sigma is unchanged (the agent retains its internal state space);
  • K∣RK|_R is the restricted skillset from Definition 3.1;
  • Π∣R(σ,u)=Π(σ,u)∩K∣R\Pi|_R(\sigma, u) = \Pi(\sigma, u) \cap K|_R restricts operator selection to K∣RK|_R.

Lemma 3.3 (Fragment identity outside region). If E∈K∣RE \in K|_R, then EE is fragment-preserving on F(H)∖R\mathcal{F}(H) \setminus R: for every fragment F∉RF \notin R, E(u)∣F=u∣FE(u)|_F = u|_F.

Proof. This is the defining property of K∣RK|_R (Definition 3.1).

Lemma 3.4 (Region restriction preserves admissibility). If E∈KE \in K is an admissible interaction operator (monotone, fragment-preserving, closure-compatible, provenance-extending, failure-monotone), then E∈K∣RE \in K|_R is also admissible.

Proof. Restricting the set of fragments an operator may modify does not weaken monotonicity, fragment-preservation, closure-compatibility, provenance extension, or failure monotonicity. Each property holds for all fragments; restricting to identity on some fragments preserves the property on the remaining ones.

4. Non-Interference Theorem

The central result: agents restricted to disjoint regions do not interfere with each other.

Theorem 4.1 (Non-interference). Let A1,A2A_1, A_2 be agents restricted to disjoint regions R1,R2R_1, R_2, with R1∩R2=∅R_1 \cap R_2 = \emptyset. Let E1∈K∣R1E_1 \in K|_{R_1} and E2∈K∣R2E_2 \in K|_{R_2}. Then:

(i) Commutativity. E1∘E2=E2∘E1E_1 \circ E_2 = E_2 \circ E_1 on U\mathcal{U}.

(ii) Admissibility of composite. The composite E1∘E2E_1 \circ E_2 is an admissible interaction operator.

(iii) Preservation of all admissibility conditions. Fragment-preservation, closure-compatibility, provenance extension, and failure monotonicity hold for E1∘E2E_1 \circ E_2.

Proof sketch.

For (i): Let u∈Uu \in \mathcal{U} and let FF be any fragment. There are three cases:

  • F∈R1∖R2F \in R_1 \setminus R_2: Then E2E_2 acts as identity on FF, so (E1∘E2)(u)∣F=E1(u)∣F(E_1 \circ E_2)(u)|_F = E_1(u)|_F. Similarly, (E2∘E1)(u)∣F=E2(E1(u))∣F=E1(u)∣F(E_2 \circ E_1)(u)|_F = E_2(E_1(u))|_F = E_1(u)|_F since E2E_2 acts as identity on FF regardless of the state.
  • F∈R2∖R1F \in R_2 \setminus R_1: Symmetric argument.
  • F∉R1∪R2F \notin R_1 \cup R_2: Both operators act as identity on FF, so both composites yield u∣Fu|_F.

Since the composites agree on every fragment, and partial states are determined by their fragment restrictions (by the sheaf condition, Theorem 15.2 of the Interactive Semioverse specification), E1∘E2=E2∘E1E_1 \circ E_2 = E_2 \circ E_1.

For (ii): Monotonicity composes. Fragment-preservation holds because each operator preserves fragments in the other’s region and its own. Closure-compatibility: for any XX,

S((E1∘E2)(X))=S(E1(E2(X)))≥E1(S(E2(X)))≥E1(E2(S(X)))\mathcal{S}((E_1 \circ E_2)(X)) = \mathcal{S}(E_1(E_2(X))) \geq E_1(\mathcal{S}(E_2(X))) \geq E_1(E_2(\mathcal{S}(X)))

where the first inequality uses closure-compatibility of E1E_1 and the second uses closure-compatibility of E2E_2 together with monotonicity of E1E_1. Provenance extension and failure monotonicity compose because each operator independently extends provenance and preserves failure marks in its region, and acts as identity elsewhere.

For (iii): This follows from (ii) and the observation that each admissibility condition is preserved under composition of operators satisfying it (established in the base Interactive Semioverse specification for monotonicity, fragment-preservation, and closure-compatibility).

5. Parallel Composition

We define a parallel composition operator for region-restricted agents.

Definition 5.1 (Parallel composition). Given agents A1∣R1,…,An∣RnA_1|_{R_1}, \ldots, A_n|_{R_n} with pairwise disjoint regions (i.e., Ri∩Rj=∅R_i \cap R_j = \emptyset for all i≠ji \neq j), define the parallel composition:

A1∥A2∥⋯∥AnA_1 \parallel A_2 \parallel \cdots \parallel A_n

as the agent whose step operator applies each Ei∈K∣RiE_i \in K|_{R_i} concurrently. Formally, the composite step operator is:

EA1∥⋯∥An(u)=(En∘⋯∘E1)(u)E_{A_1 \parallel \cdots \parallel A_n}(u) = (E_n \circ \cdots \circ E_1)(u)

where each EiE_i is selected by Πi∣Ri\Pi_i|_{R_i}.

By Theorem 4.1(i), the result does not depend on the ordering of operators: for any permutation σ\sigma of {1,…,n}\{1, \ldots, n\},

Eσ(n)∘⋯∘Eσ(1)=En∘⋯∘E1.E_{\sigma(n)} \circ \cdots \circ E_{\sigma(1)} = E_n \circ \cdots \circ E_1.

Theorem 5.2 (Parallel composition preserves admissibility). The parallel composition A1∥⋯∥AnA_1 \parallel \cdots \parallel A_n is an admissible agent: its step operator is monotone, fragment-preserving, closure-compatible, provenance-extending, and failure-monotone.

Proof. By induction on nn. The base case n=1n = 1 is trivial. For the inductive step, apply Theorem 4.1(ii)-(iii) to the composite of EnE_{n} with the operator for A1∥⋯∥An−1A_1 \parallel \cdots \parallel A_{n-1}, noting that RnR_n is disjoint from R1∪⋯∪Rn−1R_1 \cup \cdots \cup R_{n-1}.

Corollary 5.3 (Order independence). The parallel composition is well-defined up to fragmentwise extensional equality: the result depends on the set {(Ai,Ri)}\{(A_i, R_i)\} but not on any enumeration of that set.

6. Sequential Composition with Handoff

Not all multi-agent coordination involves disjoint regions. We formalize the sequential case.

Definition 6.1 (Handoff predicate). A handoff predicate is a monotone function h:U→{true,false}h : \mathcal{U} \to \{\mathrm{true}, \mathrm{false}\}: if h(u)h(u) holds and u≤u′u \leq u', then h(u′)h(u') holds. The predicate checks that a partial state satisfies the preconditions for a subsequent agent to begin work.

Definition 6.2 (Sequential composition with handoff). Given agents A1,A2A_1, A_2 and a handoff predicate hh, define the sequential composition with handoff:

A1;hA2A_1 ;_h A_2

as the agent that executes A1A_1 repeatedly until hh holds on the resulting state, then executes A2A_2. Formally, let EA1(k)E_{A_1}^{(k)} denote kk applications of A1A_1’s step operator. Define:

EA1;hA2(u,σ1,σ2)=EA2(EA1(k⋆)(u,σ1),σ2)E_{A_1 ;_h A_2}(u, \sigma_1, \sigma_2) = E_{A_2}(E_{A_1}^{(k^{\star})}(u, \sigma_1), \sigma_2)

where k⋆=min⁡{k:h(EA1(k)(u,σ1)U)=true}k^{\star} = \min\{k : h(E_{A_1}^{(k)}(u, \sigma_1)_{\mathcal{U}}) = \mathrm{true}\}, and (−)U(-)_{\mathcal{U}} projects to the state component. If no such k⋆k^{\star} exists, execution diverges (liveness failure; see Section 10).

Lemma 6.3 (Connection to work-unit chains). A sequential composition A1;hA2A_1 ;_h A_2 is a two-element chain in the work-unit model (as defined in the work-units-and-chains specification): A1A_1’s execution is the first work unit, A2A_2’s execution is the second, and hh is the token-availability check that enables the second transition.

Proof. In the Petri net model, A1A_1 is a transition whose output places include a place php_h representing the handoff condition. The predicate hh holds exactly when php_h has a token. A2A_2 is a transition with php_h as an input place. The two-element chain fires A1A_1 first (consuming input tokens, producing php_h), then fires A2A_2 (consuming php_h and producing final output tokens). This is exactly the sequential composition A1;hA2A_1 ;_h A_2.

Lemma 6.4 (Sequential composition preserves admissibility). If EA1E_{A_1} and EA2E_{A_2} are admissible interaction operators, then EA1;hA2E_{A_1 ;_h A_2} is admissible, provided k⋆k^{\star} exists.

Proof. Finite compositions of admissible interaction operators are admissible (monotonicity, fragment-preservation, closure-compatibility, provenance extension, and failure monotonicity all compose). The composite EA2∘EA1(k⋆)E_{A_2} \circ E_{A_1}^{(k^{\star})} is a finite composition.

7. Shared Configuration Artifacts

In practice, agents share configuration artifacts (e.g., cross-link-config.json). We formalize read-only shared state.

Definition 7.1 (Shared artifact). A shared artifact is a handle τconfig∈Handle\tau_{\mathrm{config}} \in \mathsf{Handle} whose fragment Fτconfig,XF_{\tau_{\mathrm{config}}, X} is designated as read-only for a set of participating agents. Formally, given a set of agents {A1,…,An}\{A_1, \ldots, A_n\} and a designated coordinator agent AcA_c:

  • Every agent AiA_i may read Fτconfig,XF_{\tau_{\mathrm{config}}, X} (include it in the interaction surface for any skill invocation);
  • Only AcA_c may modify Fτconfig,XF_{\tau_{\mathrm{config}}, X};
  • For every non-coordinator agent AiA_i (i≠ci \neq c), the policy enforces:
Πi(σ,u)∩{E∈Ki:E modifies Fτconfig,X}=∅.\Pi_i(\sigma, u) \cap \{E \in K_i : E \text{ modifies } F_{\tau_{\mathrm{config}}, X}\} = \emptyset.

This is a monotone restriction on policy: the predicate Polai(s,in,X)=false\mathrm{Pol}_{a_i}(s, \mathrm{in}, X) = \mathrm{false} whenever skill ss with input in\mathrm{in} would modify τconfig\tau_{\mathrm{config}} and aia_i is not the coordinator.

Lemma 7.2 (Shared artifacts do not break non-interference). Let A1∣R1,…,An∣RnA_1|_{R_1}, \ldots, A_n|_{R_n} be region-restricted agents sharing a read-only artifact τconfig\tau_{\mathrm{config}}. If Fτconfig,X∉RiF_{\tau_{\mathrm{config}}, X} \notin R_i for any ii (i.e., the config fragment is not in any worker’s region), then non-interference (Theorem 4.1) still holds.

Proof. Each worker’s operators act as identity on fragments outside their region, and the config fragment is outside every worker’s region. Workers read the config fragment but do not modify it. Reading is observation, not mutation: it does not change the state in U\mathcal{U}, so it introduces no conflict between operators. The commutativity proof from Theorem 4.1 applies without modification.

8. Fan-Out Pattern

The fan-out pattern is the primary parallelism mechanism in this vault. We formalize it as a composition theorem.

Definition 8.1 (Fan-out). Given:

  • a task TT specified by a skill template s∈Skilldefs \in \mathsf{Skill}^{\mathrm{def}},
  • a disjoint region cover {R1,…,Rn}\{R_1, \ldots, R_n\} of the relevant portion of F(H)\mathcal{F}(H),
  • a shared configuration artifact τconfig\tau_{\mathrm{config}},
  • a coordinator agent AcA_c and worker agents A1,…,AnA_1, \ldots, A_n,

a fan-out is the following execution:

  1. AcA_c produces τconfig\tau_{\mathrm{config}} (applying a skill that creates the configuration artifact).
  2. Each Ai∣RiA_i|_{R_i} executes ss with τconfig\tau_{\mathrm{config}} as input, modifying only fragments in RiR_i.
  3. By Theorem 4.1, steps (2) for all ii can execute in parallel.
  4. AcA_c collects effect reports from each AiA_i and produces a summary.

Formally, the fan-out is the sequential-then-parallel composition:

Acinit;hconfig(A1∣R1∥⋯∥An∣Rn);hdoneAccollectA_c^{\mathrm{init}} ;_{h_{\mathrm{config}}} (A_1|_{R_1} \parallel \cdots \parallel A_n|_{R_n}) ;_{h_{\mathrm{done}}} A_c^{\mathrm{collect}}

where hconfigh_{\mathrm{config}} checks that τconfig\tau_{\mathrm{config}} exists in the state, and hdoneh_{\mathrm{done}} checks that all workers have produced effect reports.

Theorem 8.2 (Fan-out equivalence). Let AseqA_{\mathrm{seq}} be a single agent that applies skill ss to each region RiR_i in sequence (in any order), reading τconfig\tau_{\mathrm{config}} at each step. Then the fan-out produces a result that is fragmentwise extensionally equal to AseqA_{\mathrm{seq}}:

EA1∣R1∥⋯∥An∣Rn(u)≡FEAseq(u)E_{A_1|_{R_1} \parallel \cdots \parallel A_n|_{R_n}}(u) \equiv_F E_{A_{\mathrm{seq}}}(u)

for every fragment F∈F(H)F \in \mathcal{F}(H).

Proof sketch. By Theorem 4.1(i), the parallel composition is independent of ordering. The sequential agent applies operators to disjoint regions in some order. Since each operator acts as identity outside its region (Lemma 3.3), the sequential composition in any order produces the same result as the parallel composition (each fragment is modified by at most one operator). The sheaf condition (Theorem 15.2 of the Interactive Semioverse specification) guarantees that agreement on every fragment implies agreement on the global state, so the results are fragmentwise extensionally equal.

Corollary 8.3 (Fan-out preserves admissibility). The fan-out execution is admissible: the composite operator is monotone, fragment-preserving, closure-compatible, provenance-extending, and failure-monotone.

Proof. Apply Theorem 5.2 to the parallel phase and Lemma 6.4 to the sequential phases. Composition of admissible operators is admissible.

9. Pipeline Pattern

We formalize the pipeline pattern, where the output of one skill feeds the input of the next.

Definition 9.1 (Pipeline). Given a sequence of skills s1,s2,…,sk∈Skilldefs_1, s_2, \ldots, s_k \in \mathsf{Skill}^{\mathrm{def}} such that the output schema Tout(i)T_{\mathrm{out}}^{(i)} of sis_i is compatible with the input schema Tin(i+1)T_{\mathrm{in}}^{(i+1)} of si+1s_{i+1} (there exists a monotone map ϕi:Tout(i)→Tin(i+1)\phi_i : T_{\mathrm{out}}^{(i)} \to T_{\mathrm{in}}^{(i+1)} translating outputs to inputs), define the pipeline:

s1;h1s2;h2⋯ ;hk−1sks_1 ;_{h_1} s_2 ;_{h_2} \cdots ;_{h_{k-1}} s_k

where each handoff predicate hih_i checks that sis_i has produced output satisfying ϕi\phi_i’s domain.

Each skill invocation is a work unit in the sense of the work-units-and-chains specification. The pipeline is a kk-element chain.

Lemma 9.2 (Pipeline admissibility). A pipeline s1;h1⋯ ;hk−1sks_1 ;_{h_1} \cdots ;_{h_{k-1}} s_k is admissible if each sis_i is admissible and each handoff predicate hih_i is satisfiable (i.e., there exists a state uu such that hi([ ⁣[si] ⁣](u,ini)U)h_i([\![s_i]\!](u, \mathrm{in}_i)_{\mathcal{U}}) holds).

Proof. By induction on kk, using Lemma 6.4 at each step. Satisfiability of hih_i ensures that k⋆k^{\star} exists at each handoff, preventing divergence.

Theorem 9.3 (Pipeline composite effect). The pipeline’s composite effect on partial states is the composition of individual skill effects:

[ ⁣[s1;h1⋯ ;hk−1sk] ⁣](u,in1)=([ ⁣[sk] ⁣]∘⋯∘[ ⁣[s1] ⁣])(u,in1,ϕ1,…,ϕk−1)[\![s_1 ;_{h_1} \cdots ;_{h_{k-1}} s_k]\!](u, \mathrm{in}_1) = ([\![s_k]\!] \circ \cdots \circ [\![s_1]\!])(u, \mathrm{in}_1, \phi_1, \ldots, \phi_{k-1})

where each ini+1=ϕi(outi)\mathrm{in}_{i+1} = \phi_i(\mathrm{out}_i) and outi\mathrm{out}_i is the output of sis_i.

Theorem 9.4 (Provenance trace concatenation). The composite provenance trace of a pipeline is the concatenation of individual provenance traces:

trace(s1;h1⋯ ;hk−1sk)=trace(s1)⋅trace(s2)⋯trace(sk)\mathrm{trace}(s_1 ;_{h_1} \cdots ;_{h_{k-1}} s_k) = \mathrm{trace}(s_1) \cdot \mathrm{trace}(s_2) \cdots \mathrm{trace}(s_k)

where ⋅\cdot denotes concatenation in the provenance history category P(F)P(F) (Section 9 of the Interactive Semioverse specification).

Proof sketch. Each skill invocation produces a provenance record (Section on Execution Semantics in the Agential Semioverse specification). Sequential composition concatenates records. The refinement morphisms in P(F)P(F) respect concatenation because provenance histories compose functorially (Lemma 9.1 of the Interactive Semioverse specification).

10. Coordination Correctness

We define four correctness conditions for multi-agent coordination and relate them to established Petri net properties.

Definition 10.1 (Safety). A multi-agent coordination is safe if no agent modifies fragments outside its assigned region. Formally, for each agent Ai∣RiA_i|_{R_i} and each execution step Exec(ai,s,in,u)=(u′,out,trace)\mathrm{Exec}(a_i, s, \mathrm{in}, u) = (u', \mathrm{out}, \mathrm{trace}):

for all F∉Ri,u′∣F=u∣F.\text{for all } F \notin R_i, \quad u'|_F = u|_F.

This is guaranteed by construction when agents use region-restricted skillsets (Definition 3.1).

Definition 10.2 (Liveness). A multi-agent coordination is live if, whenever all agents’ skills are admissible and all handoff predicates are satisfiable, the coordination completes in finitely many steps. No agent blocks indefinitely waiting for a resource held by another agent.

In the parallel case with disjoint regions, liveness holds trivially: agents cannot contend for shared resources, so deadlock is impossible. In the sequential case, liveness reduces to satisfiability of each handoff predicate.

Definition 10.3 (Equivalence). A multi-agent coordination is equivalent to a single-agent execution if the result is fragmentwise extensionally equal to some sequential single-agent execution. Formally, there exists an ordering σ\sigma of all skill invocations across all agents such that:

EA1∥⋯∥An(u)≡FEσ(1)∘⋯∘Eσ(m)(u)E_{A_1 \parallel \cdots \parallel A_n}(u) \equiv_F E_{\sigma(1)} \circ \cdots \circ E_{\sigma(m)}(u)

for every fragment F∈F(H)F \in \mathcal{F}(H), where Eσ(i)E_{\sigma(i)} are the individual skill invocations in order σ\sigma.

For parallel compositions with disjoint regions, equivalence holds for every ordering (Theorem 4.1). For sequential compositions, equivalence holds for the given ordering.

Definition 10.4 (Provenance completeness). A multi-agent coordination has complete provenance if every modification to the state is traced to an agent, a skill, and an input. Formally, for every fragment FF and every element a∈Fa \in F that differs between the initial and final state, there exists a provenance history in P(F)P(F) that records which agent applied which skill with which input to produce aa.

Theorem 10.5 (Correctness of parallel fan-out). A fan-out (Definition 8.1) with pairwise disjoint regions satisfies all four correctness conditions: safety, liveness, equivalence, and provenance completeness.

Proof.

  • Safety: Each worker Ai∣RiA_i|_{R_i} uses the region-restricted skillset K∣RiK|_{R_i}, which by definition acts as identity outside RiR_i (Lemma 3.3).
  • Liveness: Workers operate on disjoint regions and share no mutable state (the shared config is read-only). No contention occurs, so no deadlock is possible. The sequential phases (coordinator init and collect) complete if their skills are admissible and handoff predicates are satisfiable.
  • Equivalence: By Theorem 8.2, the parallel execution is fragmentwise extensionally equal to any sequential ordering.
  • Provenance completeness: Each worker produces an effect report (required by the Agential Semioverse specification, Section on Effect Reports). The coordinator collects these reports. Every modification is traceable to a worker, skill, and input through the individual effect reports.

Remark 10.6 (Relationship to Petri net properties). The correctness conditions correspond to standard Petri net properties as described in the petri-nets-and-lean specification:

  • Safety corresponds to boundedness: each place (fragment ownership token) has at most one token (one owning agent). The coordination is 1-bounded on ownership places.
  • Liveness corresponds to liveness in the Petri net sense: every intended transition can eventually fire. For disjoint-region parallelism, all transitions are independent (no shared input places), guaranteeing liveness. For sequential compositions, liveness reduces to reachability of the handoff marking.
  • Equivalence corresponds to confluence: different firing sequences reach the same marking. For disjoint-region parallelism, this follows from the independence of transitions sharing no input or output places.
  • Provenance completeness corresponds to observability: every transition firing is recorded. The effect report requirement of the Agential Semioverse ensures that every firing (skill invocation) produces an observable output.

11. Mixed Coordination

Real workflows combine parallel and sequential phases. We define the general composition.

Definition 11.1 (Coordination term). The set Coord\mathsf{Coord} of coordination terms is generated by:

c::=A∣R∣c1∥c2∣c1;hc2c ::= A|_R \quad|\quad c_1 \parallel c_2 \quad|\quad c_1 ;_h c_2

where A∣RA|_R is a region-restricted agent, ∥\parallel is parallel composition (requiring disjoint regions), and ;h;_h is sequential composition with handoff.

Theorem 11.2 (General coordination correctness). A coordination term c∈Coordc \in \mathsf{Coord} satisfies safety, liveness, equivalence, and provenance completeness if:

(i) Every ∥\parallel node has children with pairwise disjoint regions.

(ii) Every handoff predicate hh in a ;h;_h node is satisfiable given the preceding phase’s output.

(iii) Every leaf agent A∣RA|_R uses only admissible interaction operators.

Proof. By structural induction on cc. The base case (single agent) is given by the Agential Semioverse specification. The parallel case uses Theorem 5.2. The sequential case uses Lemma 6.4.

12. Discussion

This specification extends the Agential Semioverse with three capabilities: region-restricted execution, parallel composition over disjoint regions, and sequential composition with typed handoffs. The key insight is that the fragment lattice already provides the locality structure needed for concurrency: if two operators modify disjoint sets of fragments, they commute.

The sheaf condition on the fragment fibration (Theorem 15.2 of the Interactive Semioverse) plays a structural role: it guarantees that fragmentwise equality implies global equality, which is what lets us prove that parallel and sequential executions produce the same result.

Relationship to Claude Code’s isolation mechanisms. Claude Code provides worktree isolation: each subagent can receive a full git worktree (filesystem copy) of the repository. This is a stronger guarantee than region restriction — worktree isolation gives each agent its own complete filesystem, whereas region restriction partitions a shared filesystem. The region formalism remains valuable because:

  • It formalizes what “safe to parallelize” means for operations that do NOT use worktree isolation (e.g., multiple Agent calls operating on different directories within the same working tree).
  • It provides the mathematical basis for validation tooling that checks whether skills declare compatible regions.
  • It enables static analysis of skill compositions without running them.

Claude Code’s Agent tool also supports model selection per subagent (haiku for fast tasks, sonnet for moderate tasks, opus for hard tasks) and tool restriction via allowed_tools. These are capability constraints on the agent profile that the ASR models as the Tools component of P(a)P(a).

Three directions remain open:

  1. Overlapping regions. The non-interference theorem requires disjoint regions. In practice, some operations (e.g., cross-linking) touch fragments in multiple directory subtrees. Extending the theory to handle overlapping regions requires either a locking discipline or a merge operation on conflicting fragment modifications. The sheaf gluing condition (Theorem 15.2) provides a candidate merge semantics, but conflict resolution (when two agents produce incompatible modifications to the same fragment) requires additional structure. Claude Code’s worktree isolation sidesteps this problem by giving each agent a full copy, but the formal question remains relevant for understanding composition.

  2. Dynamic region assignment. The current formalization assigns regions statically before execution begins. A dynamic scheme, where regions are negotiated or reallocated during execution, would require extending the policy function to include region claims and a protocol for resolving competing claims. This connects to the colored Petri net extensions mentioned in the petri-nets-and-lean specification.

  3. Lean formalization. The theorems in this document admit formal proofs in Lean 4. The fragment lattice is a finite distributive lattice (a special case of a Heyting algebra); region restriction is a sublattice operation; commutativity of operators on disjoint sublattices is a standard algebraic fact. A Lean encoding would provide machine-checked guarantees that fan-out and pipeline executions preserve all admissibility conditions.
