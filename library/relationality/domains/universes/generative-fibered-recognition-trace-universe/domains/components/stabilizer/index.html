Stabilizer

2025-12-26 by gpt-5.2-codex

Concrete example first

Let Ht={⊥<m<⊤}H_t=\{\bot<m<\top\}. Define

  • σt(⊥)=⊥\sigma_t(\bot)=\bot,
  • σt(m)=⊤\sigma_t(m)=\top,
  • σt(⊤)=⊤\sigma_t(\top)=\top.

Applying σt\sigma_t once pushes uncertain recognition mm to a stable value ⊤\top. Applying it again changes nothing. That is stabilization.

Formal definition

For each trace tt, a stabilizer is an endomorphism σt:Ht→Ht\sigma_t:H_t\to H_t that is:

  • monotone: a≤b⇒σt(a)≤σt(b)a\le b \Rightarrow \sigma_t(a)\le \sigma_t(b),
  • idempotent: σt(σt(a))=σt(a)\sigma_t(\sigma_t(a))=\sigma_t(a).

In GFRTU, stabilizer maps are part of the primitive recognition dynamics and must be compatible with reindexing across traces.

Why it matters in GFRTU

  • It models convergence to stable recognitions.
  • It defines one half of the fixed-layer condition σt(a)=a\sigma_t(a)=a.
  • Together with drift, it determines the stable core that survives dynamics.
