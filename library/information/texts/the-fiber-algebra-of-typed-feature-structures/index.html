The Fiber Algebra of Typed Feature Structures

Abstract

A typed feature structure (Carpenter 1992) — the formal model of key-value data — forms a lattice under subsumption. A collection of feature structures sharing attribute names is a formal context (Ganter & Wille 1999). Belohlavek (2004) showed that when a formal context is valued in a residuated lattice LL, the resulting concept lattice inherits LL’s residuated structure. Wurm (2012) proved that concept lattices arising from distributional data are residuated lattices giving complete semantics for the full Lambek calculus. We combine these results: a collection of key-value records whose attributes carry values in the four-valued bilattice 2⊗2\mathbf{2} \otimes \mathbf{2} (Fitting 1991) generates a bilattice-valued formal context whose concept lattice is a residuated lattice with Lambek calculus structure. The residuation is the Busaniche-Cignoli (2014) adjunction, which decomposes into extending (⊙\odot) and restricting (⇒\Rightarrow). The resulting modal calculus is decidable (Rivieccio, Jung, & Jansana 2017). This applies to any system of key-value records: YAML frontmatter, JSON documents, database rows, RDF datasets.

1. Feature structures are formal contexts

1.1 Typed feature structures

Carpenter (1992) formalized key-value data as typed feature structures: a type drawn from a partially ordered set of types, together with a partial function from features (attribute names) to values. The set of all typed feature structures over a given signature forms a bounded lattice under subsumption, where F1⊑F2F_1 \sqsubseteq F_2 iff F1F_1 is at least as specific as F2F_2. Unification is meet; generalization is join.

Every key-value data format instantiates this lattice. A YAML frontmatter block is a feature structure whose type is the file and whose features are the keys. A JSON object is a feature structure whose type is the document and whose features are the property names. A database row is a feature structure whose type is the entity and whose features are the column names.

1.2 From feature structures to formal contexts

A formal context in the sense of Ganter & Wille (1999) is a triple (G,M,I)(G, M, I) where GG is a set of objects, MM is a set of attributes, and I⊆G×MI \subseteq G \times M is an incidence relation. Given a collection of feature structures sharing attribute names, the formal context is immediate: GG is the set of entities (files, documents, rows), MM is the set of attribute names (keys), and I(g,m)I(g, m) holds when entity gg has a value for attribute mm.

Pratt (1999) proved that a formal context is a Chu space over {0,1}\{0, 1\}. More generally, a many-valued formal context — where I:G×M→LI: G \times M \to L for some value set LL — is a Chu space over LL.

The derivation operators on a formal context define a Galois connection:

A′={m∈M:∀g∈A, I(g,m)}B′={g∈G:∀m∈B, I(g,m)}A' = \{ m \in M : \forall g \in A,\ I(g, m) \} \qquad B' = \{ g \in G : \forall m \in B,\ I(g, m) \}

A formal concept is a pair (A,B)(A, B) where A′=BA' = B and B′=AB' = A. The set of all formal concepts forms a complete lattice — the concept lattice B(G,M,I)\mathfrak{B}(G, M, I).

1.3 Feature fibers

For each attribute m∈Mm \in M, the feature fiber of mm is the collection of all values that mm takes across entities:

Fib(m)={(g,v):I(g,m)=v}\mathrm{Fib}(m) = \{ (g, v) : I(g, m) = v \}

When the formal context is binary (I:G×M→{0,1}I: G \times M \to \{0, 1\}), each fiber is a set. When the formal context is many-valued (I:G×M→LI: G \times M \to L), each fiber inherits the algebraic structure of LL.

2. Bilattice-valued formal contexts

2.1 Valuing attributes in 2⊗2\mathbf{2} \otimes \mathbf{2}

Belnap (1977) defined four epistemic states for multi-source information: ⊥\bot (no information), tt (told true), ff (told false), ⊤\top (told both). Fitting (1991) proved {⊥,t,f,⊤}≅2⊗2\{\bot, t, f, \top\} \cong \mathbf{2} \otimes \mathbf{2}, the twist product of the two-element Boolean algebra with itself. Fitting (2002) proved this is the free bounded distributive bilattice on one generator.

A bilattice-valued formal context is a triple (G,M,I)(G, M, I) where I:G×M→{⊥,t,f,⊤}I: G \times M \to \{\bot, t, f, \top\}. Each entity-attribute pair carries an epistemic state: the attribute is unattested (⊥\bot), affirmed (tt), denied (ff), or contested (⊤\top) for that entity.

This is the natural valuation for real key-value data. An attribute present with a value is affirmed. An attribute explicitly absent or negated is denied. An attribute with contradictory values from different sources is contested. An attribute not mentioned is unattested.

2.2 The residuated concept lattice

Belohlavek (2004) proved that when a formal context is valued in a complete residuated lattice LL, the concept lattice BL(G,M,I)\mathfrak{B}_L(G, M, I) inherits LL’s residuated structure. The Galois connection generalizes:

A′(m)=⋀g∈G(A(g)→I(g,m))B′(g)=⋀m∈M(B(m)→I(g,m))A'(m) = \bigwedge_{g \in G} (A(g) \to I(g, m)) \qquad B'(g) = \bigwedge_{m \in M} (B(m) \to I(g, m))

where →\to is the residual implication of LL.

The four-valued bilattice 2⊗2\mathbf{2} \otimes \mathbf{2} is a commutative residuated lattice (Busaniche & Cignoli 2014), with the residuated pair:

(x,y)⊙(z,v)=(x⋅z, (x→v)∧(z→y))(x, y) \odot (z, v) = (x \cdot z,\ (x \to v) \wedge (z \to y))

(x,y)⇒(z,v)=((x→z)∧(v→y), x⋅v)(x, y) \Rightarrow (z, v) = ((x \to z) \wedge (v \to y),\ x \cdot v)

satisfying a⊙p≤tqa \odot p \leq_t q iff p≤t(a⇒q)p \leq_t (a \Rightarrow q) (Chajda & Langer 2021).

Theorem 1. The concept lattice of a bilattice-valued formal context (G,M,I)(G, M, I) with I:G×M→2⊗2I: G \times M \to \mathbf{2} \otimes \mathbf{2} is a residuated lattice with the Busaniche-Cignoli residuation.

Proof. Apply Belohlavek (2004), Theorem 3.1, with L=2⊗2L = \mathbf{2} \otimes \mathbf{2}. The completeness of 2⊗2\mathbf{2} \otimes \mathbf{2} as a residuated lattice is given by Busaniche & Cignoli (2014). The Galois connection on the bilattice-valued context uses the residual ⇒\Rightarrow of 2⊗2\mathbf{2} \otimes \mathbf{2} as the →\to in Belohlavek’s generalized derivation operators. □\square

2.3 Lambek calculus structure

Wurm (2012) proved that syntactic concept lattices — concept lattices arising from distributional data over a formal language — are residuated lattices that give complete semantics for the full Lambek calculus (Lambek 1958). Galatos, Jipsen, Kowalski, & Ono (2007) established that the Lambek calculus is the logic of residuated lattices.

Theorem 2. The concept lattice of a bilattice-valued formal context has Lambek calculus structure. The product ⊙\odot is extending (combining entities through shared attributes). The implication ⇒\Rightarrow is restricting (constraining entities by attribute conditions). These are the two sides of the Lambek residuation instantiated on 2⊗2\mathbf{2} \otimes \mathbf{2}.

Proof. The concept lattice is a residuated lattice (Theorem 1). Every residuated lattice is a model of the Lambek calculus (Galatos et al. 2007, Theorem 2.20). The operations ⊙\odot and ⇒\Rightarrow are the Busaniche-Cignoli residuated pair, which is the Lambek product and its residual instantiated on 2⊗2\mathbf{2} \otimes \mathbf{2}. □\square

3. Feature fibers and the fiber functor

3.1 Fiber algebras

For each attribute m∈Mm \in M, the feature fiber Fib(m)\mathrm{Fib}(m) consists of all entity-value pairs for that attribute. The pointwise bilattice operations on Fib(m)→2⊗2\mathrm{Fib}(m) \to \mathbf{2} \otimes \mathbf{2} define a fiber algebra AmA_m.

Proposition 1. Each fiber algebra AmA_m is a distributive bilattice. (Pointwise products of distributive bilattices are distributive; Fitting 2002.)

3.2 The functor

Let A\mathcal{A} be the category whose objects are attribute names and whose morphisms are generated by attribute relationships in the data schema (subsumption, co-occurrence, functional dependency).

Theorem 3 (Fiber Functor). The assignment m↦Amm \mapsto A_m extends to a contravariant functor F:Aop→BiLat\mathcal{F}: \mathcal{A}^{\mathrm{op}} \to \mathbf{BiLat}.

Proof. Attribute subsumption (when one attribute’s values are always at least as informative as another’s) induces a restriction of valuations, preserving bilattice operations pointwise. Co-occurrence composition applies ∧k\wedge_k pointwise. Functoriality follows from the transitivity of subsumption and associativity of co-occurrence. □\square

Corollary 1. The Busaniche-Cignoli adjunction ⊙⊣⇒\odot \dashv \Rightarrow lifts to fibers: extending and restricting on each fiber algebra AmA_m are adjoint.

4. Consequences for key-value data

4.1 Decidability

Rivieccio, Jung, & Jansana (2017) proved that four-valued modal logic over 2⊗2\mathbf{2} \otimes \mathbf{2} has the finite model property and is decidable.

Corollary 2. The modal calculus over a bilattice-valued formal context — reasoning about what can be inferred by extending and restricting across key-value records — is decidable.

4.2 Linear logic interpretation

Jakl (2025) proved Chu(H,0)\mathrm{Chu}(H, 0) over a Heyting algebra HH is the twist construction. Tensor in linear logic corresponds to ⊙\odot; linear implication corresponds to ⇒\Rightarrow.

Corollary 3. Every bilattice-valued formal context is a model of linear logic, where the tracked resources are epistemic states of key-value assertions.

4.3 Post-closure

When the data system is closed under inference (propagating attribute values through entity relationships to a fixpoint, following Fitting 1991), the ⊥\bot-valued entries are resolved. The effective algebra operates on {t,f,⊤}\{t, f, \top\}.

Theorem 4. {t,f,⊤}\{t, f, \top\} is closed under ⊙\odot and ⇒\Rightarrow. The adjunction a⊙p≤tqa \odot p \leq_t q iff p≤t(a⇒q)p \leq_t (a \Rightarrow q) holds on {t,f,⊤}\{t, f, \top\}. The identity element ⊤\top is in {t,f,⊤}\{t, f, \top\}.

Proof. Exhaustive computation on all 27 triples. □\square

4.4 Application

Consider a repository of Markdown files with YAML frontmatter. Each file is an entity. Each frontmatter key is an attribute. The collection of files sharing attribute names (authors, tags, type, status, extends, requires) is a formal context in the sense of Ganter & Wille (1999).

Valuing this context in 2⊗2\mathbf{2} \otimes \mathbf{2}: a key present with a value is tt. A key explicitly absent or contradicted by another file is ff. A key with conflicting values across sources is ⊤\top. A key not mentioned is ⊥\bot.

By Theorems 1–3, this repository has a residuated concept lattice, a Lambek calculus, a fiber functor over attribute names, and a decidable modal calculus. An agent reading the repository can reason about the epistemic structure of the frontmatter — detecting contradictions (⊤\top-valued fibers), identifying gaps (⊥\bot-valued fibers after incomplete closure), and computing entailments (via the residuation) — using the algebra that is already present in the data.
