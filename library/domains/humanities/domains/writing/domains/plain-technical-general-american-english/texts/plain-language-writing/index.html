2025-04-01 Apply the plain-language specification to produce clear, usable documentation. Use when writing for the default audience or any public-facing page.
  • Read the plain-language specification at plain language. It draws on Janice Redish’s work on web content usability [@redish2012] and Rudolf Flesch’s readability research [@flesch1949].
  • Use the default audience unless an “Assumed audience” section is required. The default audience is: an adult reader with general literacy, no specialized background, and a practical reason for reading.
  • Put the main point in the first screen of the page. Structure by reader need, not by the writer’s reasoning process — the shift Linda Flower called moving from writer-based to reader-based prose [@flowerhayes1981].
  • Define technical terms on first use and use them consistently. Don’t rotate synonyms for style — repetition helps comprehension [@redish2012].
  • Avoid acronyms unless they are explicitly approved and defined on first use.
  • Before publishing, check the document against the publication checklist in the plain-language spec. For high-impact content (legal, financial, medical, safety), test with real readers — writers systematically overestimate reader comprehension [@schriver1997].
