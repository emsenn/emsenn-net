Duty of Execution

2026-04-07 The universal obligation on any agent assigned work to advance it toward completion — or to escalate if unable. The duty that makes assignment meaningful: without it, dispatch is notification without obligation.

The duty of execution is the obligation on any agent who has been assigned work to advance that work toward completion, record progress, and escalate if blocked. It is the correlative of the assigning authority’s power to assign — the assignment creates a claim in the assigner that the work will be advanced.

The five-tuple

Component Value
Bearer Any agent assigned a task
Position The assignee’s office or role
Normative system The governance framework under which the assignment was made
Required act Advance the task, record progress, escalate if blocked
Correlative party The assigning authority and the plan owner

Why this duty is universal

Every system that uses delegation to distribute work requires this duty. Without it, assignment is a broadcast with no obligation — the assigner delegates authority but the assignee owes nothing. The duty of execution is what closes the loop: the power to assign creates a liability in the assignee, and that liability IS this duty.

This is the formal content of what Barnard (The Functions of the Executive, 1938) calls the “zone of indifference” — the range of acts an organizational member will accept as legitimate assignments. The duty of execution defines the obligation side of that acceptance.

Discharge

The duty is discharged when the task reaches a terminal state (completed, failed, or abandoned) with the path documented. Partial performance (task in progress) is not discharge — the duty persists until termination. This follows from the discharge condition: the performance act must be doubly settled (a∈Ht∗a \in H^*_t), and an in-progress task is by definition not yet settled.

Three failure modes

  1. Silent drop — task assigned, agent never engages. No events, no status change.
  2. Unrecorded progress — work happens but no record. The tracking system cannot see it.
  3. Unescalated blockage — agent is blocked but does not report. The assigning authority cannot unblock what it doesn’t know is blocked.
