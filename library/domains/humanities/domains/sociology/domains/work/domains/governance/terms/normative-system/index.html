Normative System

Formal tuple

A NormativeSystem is a quintuple N=⟨Ag,R,Sc,Π,Δ⟩N = \langle \mathrm{Ag}, R, \mathrm{Sc}, \Pi, \Delta \rangle:

N=(Ag,  R,  Sc,  Π:Sc×Sc→{0,1},  Δ⊆{O(φ),P(φ),F(φ)})N = (\mathrm{Ag},\; R,\; \mathrm{Sc},\; \Pi : \mathrm{Sc} \times \mathrm{Sc} \to \{0,1\},\; \Delta \subseteq \{O(\varphi), P(\varphi), F(\varphi)\})

where:

  • Ag\mathrm{Ag} — a set of agent types that can participate
  • RR — a set of roles: normative positions agents adopt on entry to a scene (interface types specifying deontic constraints for the duration of participation; see Role)
  • Sc\mathrm{Sc} — a set of scenes: structured interaction contexts, each governed by a protocol
  • Π\Pi — the performative structure: a directed graph on Sc\mathrm{Sc} specifying permitted scene transitions (Π(s1,s2)=1\Pi(s_1, s_2) = 1 iff scene s2s_2 may follow s1s_1 under some condition)
  • Δ\Delta — the norm set: deontic constraints over agent actions within each scene:
    • O(φ)O(\varphi) — obligation: the agent must produce φ\varphi
    • P(φ)P(\varphi) — permission: the agent may produce φ\varphi without violating a norm
    • F(φ)F(\varphi) — prohibition: the agent must not produce φ\varphi

The performative structure Π\Pi. The directed graph Π\Pi on Sc\mathrm{Sc} corresponds to the stepping relation on TT: Π(s1,s2)=1\Pi(s_1, s_2) = 1 if and only if s2∈stepT(s1)s_2 \in \mathrm{step}_T(s_1) — scene s2s_2 is a one-step extension of scene s1s_1. The set of permitted trajectories through Π\Pi is the set of histories in TT that respect the scene transition rules, which is exactly the site’s coverage structure.

What this is

A NormativeSystem is a tuple N = ⟨Ag, R, Sc, Π, Δ⟩ governing agent interaction within an Institution:

  • Ag: a set of agent types that can participate in the system
  • R: a set of roles — normative positions agents can adopt on entry
  • Sc: a set of scenes — structured interaction contexts, each governed by a protocol
  • Π: the performative structure — a directed graph over Sc defining permitted scene trajectories (which scenes can follow which, and under what conditions)
  • Δ: the norm set — deontic constraints O(φ), P(φ), F(φ) (obligations, permissions, prohibitions) on agent actions within each scene

Agents are external to the normative system until they adopt a role. Role adoption is the act of entering the institutional frame and accepting its deontic constraints for the duration of scene participation. The normative system does not control agents directly; it specifies norms that agents may follow or violate, with enforcement mechanisms as a separate component.

Scenes and protocols

A scene is a structured interaction context with a defined protocol: a set of possible moves, a set of roles, and rules governing move legality given the interaction history so far. A scene protocol is an Arena with deontic layering: the arena specifies legal moves; the protocol adds obligations (what an agent in a role must do), permissions (what an agent may do without violating the norm), and prohibitions (what an agent must not do).

The performative structure Π\Pi connects scenes: it defines when one scene can be entered as a consequence of another’s completion, what role assignments persist across scene boundaries, and what information passes between scenes. Π\Pi is a directed graph whose nodes are scenes and whose edges are performative transitions — analogous to the enabling relation in an Arena but operating at the scene level rather than the move level.

Roles and role adoption

A role in a normative system is not the same as an office in the Officer sense. A role in a normative system is an interface type: a named set of deontic permissions and obligations that apply to an agent for the duration of a scene or session. An office (in the Hohfeldian sense) is a persistent deontic binding created by a constitutive act of installation. The role/office distinction tracks the difference between a scene-local normative constraint and a persistent institutional position.

Role adoption in a normative system is typically voluntary and revocable: an agent enters a scene by committing to a role, and exits when the scene ends. Office installation creates a persistent binding across scenes and sessions.

Normative system vs. institution

A normative system is the computationally precise formalization of an Institution. The correspondence:

Institution (Searle) Normative System (MAS)
Constitutive rules Scene protocols and role definitions in R, Sc
Status-function declaration Role adoption act
Deontic powers Norm set Δ
Collective recognition Role adoption + enforcement mechanism
Background conditions Agent types Ag and their capabilities

Not every institution is fully captured by a normative system. In particular, Hodgson’s reconstitutive downward causation — the way institutions reshape agent dispositions over time — is not modeled in the tuple NN, which treats agent types Ag\mathrm{Ag} as fixed.

Normative system vs. arena

An Arena is a normative system stripped to its enabling relation: only which moves are legal after which, with no deontic layering. An arena has no obligations, no permissions beyond legality, no performative structure above the move level. A normative system adds: role assignments, obligations and prohibitions beyond mere legality, scene composition, and norm enforcement. Every scene protocol in a normative system has an underlying arena; not every arena is a scene protocol (arenas have no deontic operators).

Separation is not a norm axiom. The classical view of a normative system requires that norms be uniquely determined — that two norm instances that agree on every sub-scene must be equal. In RelationalUniverseGeometricTheory, this uniqueness clause is absent from the axioms: the Glue sequent (Group 7) is existential (there exists an amalgamation) but not separated (uniqueness requires a universal quantifier, which is not geometric). This is correct: two norm instances that agree on every sub-scene but disagree globally are shadow elements of each other, and both are resolved by the generation map RelationalHistoryFiberNuclearQuartetExtremallySlowGrowthGenerationMap to a unique element of RelationalHistoryFixedFiber at depth one higher. Non-separated norms are not a failure of the normative system — they are norms in the process of settlement. The nuclear quotient (applying RelationalHistoryFiberSaturatingNucleus) produces the unique canonical settled version. Separation is a consequence of norm settlement, not a primitive norm axiom.

Open questions

  • Whether the performative structure Π over scenes has a direct categorical description — whether it is a category itself (scenes as objects, performative transitions as morphisms) or a directed graph in some richer structure.
  • Whether the norm set Δ can be modeled as a set of Lawvere-Tierney topologies on the scene-level topos — one topology per deontic modality (O, P, F).
  • How to formally represent norm violation and enforcement within the normative system tuple, and whether the nucleus fixed-point structure provides the right model for “what has been normatively settled”.
