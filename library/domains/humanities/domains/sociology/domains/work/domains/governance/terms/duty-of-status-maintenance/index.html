Duty of Status Maintenance

2026-04-07 The obligation to maintain accurate, current status on work entities — ensuring that the recorded state matches the actual state. The duty that makes tracking meaningful: without it, records drift from reality.

The duty of status maintenance is the obligation to keep records accurate. Every work entity — plans, tasks, projects — has a status. The status must reflect reality. When status drifts from reality, every system that reads it (dispatch, briefing, reporting, review) operates on fiction.

¶The five-tuple

Component Value
Bearer The office responsible for operational tracking
Position The tracking office
Normative system The governance framework’s commitment to visible labor
Required act Every work entity has a current, accurate status; every change is recorded as an event
Correlative party All offices that read work status (dispatchers, briefers, reviewers, plan owners)

¶Why this is a distinct duty

The duty of execution says: do the work. The duty of plan stewardship says: ensure the plan converges. This duty says: ensure the RECORD of work and plan status is accurate. These three duties can fail independently:

  • Work advances but status isn’t updated (execution without maintenance — invisible labor)
  • Status is updated but work hasn’t actually advanced (maintenance without execution — fiction)
  • Plan converges but individual task statuses are stale (stewardship without maintenance — unclear picture)

¶Discharge

Discharged when zero drift exists between actual state and recorded state. This is a continuous duty — unlike execution (which terminates when the task ends), status maintenance persists as long as work entities exist.
