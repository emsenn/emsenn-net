2026-03-29 An episode of doing with a defined output — the base concept of which all work types are specializations.

Work is an act with a defined output. It restricts the general concept of action — what a thing does that produces change — to episodes with nameable results. An act that produces nothing nameable is not work; it is time spent. Work inherits from act the structure of capacity realization: something has the potential to produce an output, and work is the episode where that potential is realized.

This is the base concept. All organized activity is work when it has a defined output. The four types of work — process, procedure, derivation, and inquiry — are specializations based on which qualities of the action are settled. Two qualities matter: whether the meaning of the output is known (meaning-settled) and whether the execution path to produce it is known (execution-settled). Process has both settled. Inquiry has neither. Procedure and derivation each have one.

Work type Meaning settled Execution settled
Process yes yes
Procedure yes no
Derivation no yes
Inquiry no no

A work unit is a concrete instance of work: it names what it acts on (domain), which aspect changes (affected), how the change is produced (operation), and the specific values that make this instance distinct from other instances of the same operation (parameters). A work unit without parameters is a template, not an instance.

Work composes. A workflow is a set of work units with the constraint that the initial input type equals the final output type — the workflow takes something of type X and produces something of type X. A task is a named planning container for work units with a completion condition. Workflows are algebraic (they compose); tasks are organizational (they group work for planning and tracking).

Work is the thing that practice does and method governs. Practice without work is idle; method without work is shelfware; work without method is improvisation; work without practice is a plan that never executes.
