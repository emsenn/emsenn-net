Process

A Process is a fully determined operation: both its meaning and its execution path are fixed before it runs. Same input always produces same output.

A process is a defined, repeatable transformation: given the same input, it always produces the same output. No decisions, no branches, no judgment. The specification fully determines the behavior.

This is what “process” means in business process management (ISO 9000), formal methods, and manufacturing: a transformation whose complete behavior is determined by its specification before it runs. Baking a recipe with exact measurements. Running a build script. Executing a SQL query. Multiplying two matrices. In each case, the input and the specification together determine the output.

For the general concept — grounded in sociology and organizational theory — see process in the work taxonomy. That file defines process as one of four work types classified by whether meaning and execution are settled. This file is the FARS formalization: how the general concept maps to the relational framework’s algebraic structures.

¶The four work types

A process sits in one position of a 2×2 classification based on what is settled:

Execution settled Execution open
Meaning settled Process — fully determined Procedure — branches on conditions
Meaning open Derivation — follows rules, meaning emerges Inquiry — open exploration

A process is the doubly-settled position: both meaning (what it produces) and execution (how it proceeds) are fixed.

¶FARS algebraic reading

In the relational framework, this classification maps to fixed-point conditions on two closure operators (nuclei) in a Heyting algebra fiber HtH_t:

  • σt\sigma_t (the saturating nucleus) — closure under meaning. σt(W)=W\sigma_t(W) = W means the operation’s meaning is settled.
  • Δt\Delta_t (the transferring nucleus) — closure under execution. Δt(W)=W\Delta_t(W) = W means the execution path is settled.

A process is an element satisfying both:

W∈Ht∗=Fix(σt)∩Fix(Δt)W \in H^*_t = \mathrm{Fix}(\sigma_t) \cap \mathrm{Fix}(\Delta_t)

The joint projection πt=σt∘Δt:Ht→Ht∗\pi_t = \sigma_t \circ \Delta_t : H_t \to H^*_t maps any work element into the process position — closing all open questions. A process is already at the fixed point.

As a Skill type, a process is a morphism f:A→Bf : A \to B where f(a)∈Ht∗f(a) \in H^*_t for all inputs — the output is always fully settled.
