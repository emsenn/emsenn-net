Natural Transformations

2025-12-26

¶Entry conditions

Use natural transformations only when you already have:

  • Two functors F,G:C→DF, G:C \to D between the same categories.
  • A morphism in DD for each object of CC.
  • A requirement that these morphisms are compatible with all morphisms in CC.

¶Definitions

  • A natural transformation η:F⇒G\eta:F \Rightarrow G assigns to each object AA in CC a morphism ηA:F(A)→G(A)\eta_A: F(A) \to G(A) in DD, such that for every morphism f:A→Bf:A \to B in CC, G(f)∘ηA=ηB∘F(f). G(f) \circ \eta_A = \eta_B \circ F(f).

¶Vocabulary (plain language)

  • Component: the specific morphism ηA\eta_A for an object AA.
  • Naturality: the condition that the square above commutes for every morphism.

¶Symbols used

  • η:F⇒G\eta:F \Rightarrow G: a natural transformation.
  • ηA\eta_A: the component at AA.

¶Intuition

A natural transformation is a systematic way to transform one functor into another without breaking structure. It says that the transformation is compatible with every morphism in the source category.

¶Worked example

Let F,G:Set→SetF,G:\mathbf{Set} \to \mathbf{Set} both map a set to a product with a fixed set. The projection maps form a natural transformation because they commute with all functions between sets.

¶How to recognize the structure

  • You can assign a component morphism for every object.
  • The naturality square commutes for every morphism.

¶Common mistakes

  • Defining components without checking naturality.
  • Using different categories for FF and GG.

¶Minimal data

  • Two functors FF and GG with the same source and target.
  • A component morphism for each object.
  • A proof of the naturality condition.
