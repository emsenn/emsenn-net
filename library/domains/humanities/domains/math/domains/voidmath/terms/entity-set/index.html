Entity set

2026-04-29 The set whose elements are entities — the source of the cell index E × E over which predication fiber algebras are built. Carries no required structure beyond set-hood; specific applications may add finiteness, sortedness, or other refinements.

An entity set, denoted EE, is a set whose elements are entities. The entity set is one of the base parameters of the predication framework, alongside a predicate set Π\Pi (the object set of a predicate category) and a residuated lattice L\mathbf{L} from which the bilattice is built.

The entity set carries no required structure beyond set-hood in general. Specific theorems and applications may impose additional structure:

  • Finiteness. Many constructions (the predicate-category generator-exhaustiveness theorem, the existence of certain limits, decidable settlement checks) require EE to be finite, often with ∣E∣≥3|E| \geq 3 for the equivariance hypothesis to be non-degenerate.
  • Sortedness. EE may be sorted into kinds: E=⨆iEiE = \bigsqcup_i E_i where each EiE_i is a sub-set of entities of one type. Sortedness affects which predicates apply to which entity pairs.
  • Group action. The framework’s Sym(E)\mathrm{Sym}(E)-equivariance hypothesis treats EE as a set acted on by its symmetric group Sym(E)\mathrm{Sym}(E); this is what makes the four predicate-category generator classes exhaustive (under ∣E∣≥3|E| \geq 3).

¶The cell index

The Cartesian product E×EE \times E is the cell index — the set of pairs over which predication fiber algebras are built. For each predicate pp, the fiber algebra ApA_p assigns a value to each pair (s,o)∈E×E(s, o) \in E \times E, where ss is the subject and oo is the object of the resulting predication cell.

The cell index is the same for every predicate; what varies between predicates is the morphisms of the predicate category, not the index. This uniformity is what makes the predication framework cleanly fibrational.
