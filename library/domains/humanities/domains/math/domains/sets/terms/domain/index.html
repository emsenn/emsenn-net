2025-12-26 The set of allowed inputs to a function or relation — where the mapping starts.

The domain of a function f:A→Bf: A \to B is the set AA — the collection of all allowed inputs. Every element of the domain must be assigned exactly one output. A function is not permitted to leave any input unaccounted for. This totality requirement — that ff be defined on every element of its domain — is part of what distinguishes a function from a partial function or a relation.

The domain is not a byproduct of the function’s rule; it is part of the function’s specification. Two functions with the same formula but different domains are different functions. The function f(x)=x2f(x) = x^2 with domain N\mathbb{N} (the natural numbers) is a different function from g(x)=x2g(x) = x^2 with domain R\mathbb{R} (the real numbers), even though they agree on every input where both are defined. They have different domains, and therefore different properties: ff is a function from N\mathbb{N} to N\mathbb{N}, while gg is a function from R\mathbb{R} to R\mathbb{R}. The first can never take a negative input; the second can.

Specifying a function completely requires three things: its domain, its codomain, and the assignment rule. Leaving out the domain is like leaving out the subject of a sentence — the function is not fully determined. In practice, when mathematicians write “f(x)=xf(x) = \sqrt{x}” without specifying a domain, they implicitly mean the largest set on which the rule makes sense (the nonnegative reals, in this case). But this convention hides a choice, and in rigorous work the domain must be stated.

The domain of a binary relation R⊆A×BR \subseteq A \times B is the set AA from which the first components of the ordered pairs are drawn. Unlike a function, a relation does not require every element of AA to participate — some elements of AA may not appear in any pair.

In a category, the domain of a morphism f:A→Bf: A \to B is the source object AA. Every morphism has a domain and a codomain. Composition g∘fg \circ f is defined only when the codomain of ff equals the domain of gg — this matching condition is the basic constraint that governs which morphisms can be chained together, and it is the categorical analogue of the requirement that a function’s output land in the next function’s input set.
