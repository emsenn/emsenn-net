2025-12-26 A pair of objects where order matters — the building block of Cartesian products, binary relations, and functions.

An ordered pair (a,b)(a, b) is a pair of objects where order matters. The pair (3,5)(3, 5) is different from (5,3)(5, 3). This is the entire point: a plain set {3,5}\{3, 5\} is the same as {5,3}\{5, 3\} — sets do not track order — but an ordered pair does. The first component and the second component are distinguished positions, and swapping them produces a different pair (unless both components are equal).

Two ordered pairs are equal if and only if their components match position by position: (a,b)=(c,d)(a, b) = (c, d) if and only if a=ca = c and b=db = d. This equality condition is the defining property of ordered pairs, and it is what makes them useful for representing relationships between objects.

In set theory, ordered pairs must be built from sets, because set theory has no primitive notion of “order.” Kazimierz Kuratowski gave the standard construction in 1921: (a,b)={{a},{a,b}}(a, b) = \{\{a\}, \{a, b\}\}. This encoding looks peculiar — it represents the pair as a set of sets — but it works: from the axioms of set theory, you can prove that {{a},{a,b}}={{c},{c,d}}\{\{a\}, \{a, b\}\} = \{\{c\}, \{c, d\}\} if and only if a=ca = c and b=db = d. The construction is not the only possible one (Norbert Wiener gave an earlier but more cumbersome version in 1914), but Kuratowski’s is the simplest and has become standard.

The Cartesian product A×BA \times B is the set of all ordered pairs (a,b)(a, b) with a∈Aa \in A and b∈Bb \in B. If AA has 3 elements and BB has 4, then A×BA \times B has 3×4=123 \times 4 = 12 ordered pairs. The Cartesian product R×R=R2\mathbb{R} \times \mathbb{R} = \mathbb{R}^2 is the plane: each point is an ordered pair of real numbers (x,y)(x, y), and the order distinguishes the horizontal coordinate from the vertical coordinate.

Ordered pairs are the building blocks of binary relations and functions. A binary relation from AA to BB is a subset of A×BA \times B — a collection of ordered pairs specifying which elements of AA are related to which elements of BB. The relation “less than” on {1,2,3}\{1, 2, 3\} is the set {(1,2),(1,3),(2,3)}\{(1, 2), (1, 3), (2, 3)\}. A function f:A→Bf: A \to B is a binary relation where each element of AA appears as the first component of exactly one pair — every input has exactly one output. Without ordered pairs, there would be no functions, and without functions, most of mathematics could not be stated.

Ordered nn-tuples (a1,a2,…,an)(a_1, a_2, \ldots, a_n) extend the idea to any finite length. A triple is an ordered pair where one component is itself a pair: (a,b,c)=((a,b),c)(a, b, c) = ((a, b), c). This nesting lets you build tuples of any length from pairs alone — the pair is the universal building block for ordered sequences.

In a category, the Cartesian product generalizes to the categorical product, defined by a universal property (a pair of projection morphisms satisfying a uniqueness condition) rather than by element-level construction. The set-theoretic and categorical definitions agree in Set and can diverge in other categories, where “elements” may not be available.
