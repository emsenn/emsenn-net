2026-03-30 The basic structure of an assertion: a subject, a predicate relating it to an object, and the object. The atomic unit of knowledge representation.

A subject-predicate-object triple (SPO triple) is the basic structure of an assertion: a subject SS, a predicate PP that relates it, and an object OO. “The file extends legibility” has subject “the file,” predicate “extends,” and object “legibility.”

Frege (1879) established the logical form: the predicate is an unsaturated function that takes the subject and returns a value. The SPO triple is function application — P(S)=OP(S) = O.

The structure appears across disciplines under different names:

  • Linguistics: subject-verb-object sentence structure
  • Semantic web: RDF triples (W3C 2024) — the atomic unit of linked data
  • Databases: entity-attribute-value (EAV) — a row, a column name, and a cell value
  • Logic: a relation P(S,O)P(S, O) applied to two terms
  • Key-value data: a file (subject) with a key (predicate) mapped to a value (object)

Lambek (1958) formalized how the predicate combines with its arguments as a residuation: the product and its two residuals exhaust the modes of functor-argument interaction. The predicate’s combination with its arguments is algebraically determined.

When SPO triples carry values from the four-valued bilattice {⊥,t,f,⊤}\{\bot, t, f, \top\}, each assertion has an epistemic state: unattested, affirmed, denied, or contested. Shared predicates across triples generate fibers whose algebra is a concept lattice with Lambek calculus structure.
