Categories

2025-12-26

¶Entry conditions

Use a category only when you can:

  • Identify objects and morphisms between them.
  • Define composition of morphisms that is associative.
  • Provide identity morphisms for every object.

If composition is undefined or not associative, do not use category theory.

¶Definitions

  • A category has objects, morphisms, identities, and associative composition.

¶Vocabulary (plain language)

  • Object: a thing you care about (a set, state, type, or context).
  • Morphism: a directed arrow or process from one object to another.
  • Composition: doing one morphism after another.
  • Identity morphism: the “do nothing” morphism for an object.

¶Symbols used

¶Symbols used

  • f:A→Bf:A \to B: a morphism from object AA to object BB.
  • g∘fg \circ f: compose ff then gg.
  • idA\mathrm{id}_A: the identity morphism on AA.

¶Intuition

Categories are about processes between things, not just the things themselves. Objects can be types, states, or contexts. Morphisms are the allowed transformations. Composition means “do this, then that,” and associativity means the grouping does not matter.

¶What the axioms mean

  • Identity: every object has a “do nothing” morphism.
  • Associativity: if ff, gg, hh can compose, then (h∘g)∘f=h∘(g∘f)(h \circ g) \circ f = h \circ (g \circ f).

¶Worked examples

¶Example 1: A category from sets

Objects are sets. Morphisms are functions between sets. Composition is function composition. Identities are identity functions. This is the category Set.

¶Example 2: A category from a partial order

Given a partial order (S,≤)(S,\leq), make a category where there is a morphism a→ba \to b exactly when a≤ba \leq b. Composition exists because ≤\leq is transitive.

¶How to recognize the structure

  • Can you compose your transformations? Is it associative?
  • Does each object have an identity transformation?

¶Common mistakes

  • Treating informal “influences” as morphisms without explicit composition.

¶Minimal data

  • A list of objects.
  • A list of morphisms with source and target.
  • A composition rule and identity for each object.

¶Misuse warnings

  • Do not treat “influences” as morphisms unless composition is explicit.
  • Do not use functors if mapping does not preserve identities and composition.
