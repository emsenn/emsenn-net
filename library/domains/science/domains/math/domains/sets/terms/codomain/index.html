2025-12-26 The set of possible outputs of a function or relation — where the mapping lands.

The codomain of a function f:A→Bf: A \to B is the set BB — the collection of possible outputs. It is the set that the function maps into, though it need not map onto: not every element of the codomain is required to actually appear as an output. The elements of BB that do appear as outputs form the image (or range) of ff, which may be a proper subset of BB.

The distinction between codomain and image is one of the most commonly confused points in elementary mathematics. The function f(x)=x2f(x) = x^2 with domain R\mathbb{R} maps every real number to a nonnegative real number. If the codomain is declared as R\mathbb{R}, then f:R→Rf: \mathbb{R} \to \mathbb{R} is not surjective — negative numbers are in the codomain but are never hit. If the codomain is declared as [0,∞)[0, \infty), then g:R→[0,∞)g: \mathbb{R} \to [0, \infty) with the same rule is surjective — every nonnegative number is the square of something. These are different functions, even though they assign the same output to every input, because they have different codomains. Surjectivity — whether the image fills the codomain — depends on the codomain, and the codomain is a choice, not a consequence of the rule.

This is why specifying a function requires three things: domain, codomain, and assignment rule. The codomain is part of the function’s identity, just as much as the domain is. A function f:A→Bf: A \to B and a function g:A→Cg: A \to C with B≠CB \neq C are different functions even if they happen to give the same output for every input (which can happen when the image is a subset of both BB and CC).

The codomain of a binary relation R⊆A×BR \subseteq A \times B is the set BB from which the second components of the ordered pairs are drawn.

In a category, the codomain of a morphism f:A→Bf: A \to B is the target object BB. Composition g∘fg \circ f is defined only when the codomain of ff matches the domain of gg: the output of the first morphism must land in the input set of the second. This matching condition is the basic compositional constraint of category theory, and it generalizes the elementary fact that to chain functions together, the output of one must be a valid input to the next.
