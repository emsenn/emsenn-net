2026-03-30 A triple (G, M, I) of objects, attributes, and an incidence relation — the input structure for formal concept analysis.

A formal context (Ganter & Wille 1999) is a triple (G,M,I)(G, M, I) where GG is a set of objects, MM is a set of attributes, and I⊆G×MI \subseteq G \times M (or I:G×M→LI: G \times M \to L for a many-valued context) is an incidence relation.

I(g,m)I(g, m) states that object gg has attribute mm. In the binary case, this is yes/no. In the many-valued case, I(g,m)I(g, m) takes a value from a structured set LL.

The derivation operators on a formal context define a Galois connection between subsets of objects and subsets of attributes. The fixed points of this connection — pairs (A,B)(A, B) with A′=BA' = B and B′=AB' = A — are the formal concepts. They form a concept lattice.

A formal context is a Chu space over the value set (Pratt 1999). Binary formal contexts are Chu spaces over {0,1}\{0,1\}.

Any collection of key-value records sharing attribute names is a formal context: records are objects, attribute names are attributes, values give the incidence. A repository of YAML files with frontmatter is a formal context where the files are objects and the frontmatter keys are attributes.

When the incidence is valued in a residuated lattice LL, the concept lattice inherits LL’s residuated structure (Belohlavek 2004). When LL is the four-valued bilattice, the concept lattice has Lambek calculus structure.
