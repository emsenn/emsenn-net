first-class function

2026-03-05
Defines first-class function

A first-class function is a function that a language treats as a value: it can be stored in a variable, passed as an argument to another function, and returned as a result from a function. The function itself — not its name or its result — is the value that moves through the program.

This property is what makes higher-order functions possible. If functions are not first-class, then map, filter, and reduce cannot exist in their general form, because there is no way to hand a function to another function as an argument.

In languages with first-class functions, a function literal (often called a lambda expression or anonymous function) can be written inline wherever a value is expected. In Python, lambda x: x + 1 produces a function value without giving it a name. In JavaScript, (x) => x + 1 does the same. These anonymous functions are values like any other — they can be assigned, passed, or returned.

First-class functions are not exclusive to functional programming. Most modern languages — Python, JavaScript, Ruby, Rust, Swift — treat functions as first-class values. What distinguishes functional programming is not that functions are first-class, but that the entire paradigm is organized around composing them.

The concept has a precise mathematical origin. In the lambda calculus, every expression is a function, an application of a function, or a variable. There is no separate category for “data” versus “code.” Functions are the only kind of value. Languages that treat functions as first-class are, in this sense, reflecting the structure of the lambda calculus.
