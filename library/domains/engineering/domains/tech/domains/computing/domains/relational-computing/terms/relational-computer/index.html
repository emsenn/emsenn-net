Relational Computer

2026-04-27 A computer whose primary substrate is the relation: data is encoded as associations between entities, and computation is the addition, removal, and traversal of those associations in a graph.

A relational computer is a computer whose primary substrate is the relation: data is encoded as associations between entities, and computation is the addition, removal, and traversal of those associations in a graph. The relational computer takes relationality as its organizing ontology — an entity is constituted by the relations it participates in, not by its location in memory or the value stored at that location. Two entities are the same when their relational structure is the same; an entity persists when its relations persist; an entity is observed by traversing the edges incident to it.

A query is a graph traversal: starting from an entity, one follows edges of named predicates to reach related entities. There is no fetch-from-address primitive — the closest analog is “what is related to entity e by predicate p?”, a question the graph answers by enumeration. A write is the addition or retraction of a relation; there is no value-mutation primitive, because there are no stored values to mutate.

A relational computer contrasts with a boolean computer — one whose data is two-valued and whose computation is the manipulation of those values held at memory addresses. Most architectures realized in physical hardware to date are boolean computers. The substrate choice is ontological, not architectural: layering a graph database on a boolean-computer store does not produce a relational computer, because storage, addressing, and operations still presume the boolean substrate underneath.
