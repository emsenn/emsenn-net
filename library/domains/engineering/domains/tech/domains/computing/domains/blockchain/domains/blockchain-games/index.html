Blockchain Games

Blockchain games are virtual worlds where game state, rules, and ownership are recorded on public blockchains like Ethereum. This differs fundamentally from traditional games where a developer controls the server, data, and rules.

The field emerged from two insights:

  1. Smart contracts are deterministic state machines — they can enforce game rules transparently and immutably
  2. Blockchains are public, verifiable databases — they solve the trust problem: no one can cheat or modify history

Core Concepts

  • Entity Component System — The architectural pattern separating data from behavior, used in game engines and now in blockchain frameworks
  • Zero-Knowledge Proof — Cryptographic technique enabling private state on public blockchains, used for fog of war

Key Texts

  • Autonomous Worlds Concept — The vision of persistent, permissionlessly-extensible virtual environments
  • MUD Framework Architecture — The primary framework for building on-chain games; deep technical dive into Store, World, Tables, Systems
  • Dark Forest: Zero-Knowledge Games — The pioneering on-chain RTS that demonstrated private state via zkSNARKs
  • MUD-Based Games — Survey of production games built on MUD: OPCraft (voxel world), Sky Strife (RTS), and others

Notable Games and Projects

  • Dark Forest — First fully on-chain RTS; uses zkSNARKs for private planet locations; still actively played and community-forked
  • OPCraft — Lattice’s Minecraft-like game on Optimism; demonstrated permissionless composability and emergent governance
  • Sky Strife — Lattice’s on-chain RTS on Redstone; shows real-time gameplay at scale
  • Redstone Network — EVM-compatible rollup optimized for autonomous worlds; lower gas costs and higher throughput than Ethereum
  • Biomes — Community voxel sandbox on Redstone
  • Imminent Solace — Battle royale with zero-knowledge fog of war

Research Questions

  • How much throughput is sufficient for different game genres? (Turn-based vs. real-time)
  • What on-chain economies sustain engagement? (Play-to-earn, cosmetics, or ownership?)
  • Can AI agents play autonomously within blockchain games? (And should they be allowed?)
  • How does community governance change when rules are code instead of policy?
