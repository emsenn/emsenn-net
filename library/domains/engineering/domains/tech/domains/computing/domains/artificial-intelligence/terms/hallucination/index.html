Hallucination

2026-03-29 The generation of text by a language model that is fluent and plausible but factually false — a consequence of the model optimizing for statistical coherence rather than truth.

A hallucination is text generated by a large language model that is fluent and plausible but factually false. The model produces false statements with the same confidence as true ones because its generation mechanism — predicting the next token based on statistical patterns — optimizes for coherence, not truth. A model trained on text about historical events generates text that reads like history but may contain invented dates, fabricated quotes, or nonexistent sources.

Hallucination is a structural property of the generation process. A transformer produces each token by sampling from a probability distribution over its vocabulary, conditioned on the preceding tokens. Nothing in this process distinguishes true statements from false ones — both are sequences of tokens with high conditional probability. The model has no internal representation of truth; it has a representation of what text looks like.

Categories

Factual hallucination. The model states a false fact: a wrong date, a nonexistent paper, a fabricated statistic. This is the most commonly discussed form.

Fabricated references. The model invents citations — plausible-looking author names, journal titles, and DOIs that do not correspond to real publications. This is a specific and pervasive form of factual hallucination.

Intrinsic hallucination. The model contradicts information provided in its own context — the user gives a number, and the model uses a different number in its response.

Extrinsic hallucination. The model adds information that is neither supported nor contradicted by its context — plausible-sounding details that the source material does not contain.

Mitigation

No technique eliminates hallucination entirely. The primary mitigations reduce its frequency or detect it after generation:

  • Retrieval-augmented generation grounds the model’s output in retrieved source documents. The model is more likely to reproduce facts from documents in its context window than to fabricate them from memory. RAG shifts the dominant failure mode from fabrication to misreading.
  • Structured output with schema validation catches hallucinations at the structural level — a hallucinated tool call with a wrong field name fails validation and triggers a retry.
  • RLHF trains models to prefer truthful and grounded responses. This reduces hallucination rates but does not eliminate them.
  • Chain-of-thought prompting and self-consistency (generating multiple responses and checking for agreement) provide partial detection.

Relevance to agentic systems

Hallucination matters more in agentic systems than in conversational use because agents act on their outputs. A hallucinated fact in a conversation is a wrong answer. A hallucinated tool call in an agent is a wrong action — a file written with incorrect content, an API called with fabricated parameters, a plan based on a nonexistent capability. Human-in-the-loop review and structured output validation are the primary defenses.
