OpenClaw Gateway

2026-03-09 The long-running control-plane process in an OpenClaw deployment.

The OpenClaw Gateway is the long-running control-plane process in an OpenClaw deployment. It coordinates sessions, channels, tools, configuration, routing, and web interfaces while the assistant remains the user-facing product surface.

Role

The Gateway gives OpenClaw a persistent runtime. It keeps the assistant reachable through chat channels, the Control UI, WebChat, and paired device surfaces without reducing the system to a single interactive terminal session.

Trust boundary

OpenClaw’s security model treats the Gateway as a trusted operator boundary, not as a hostile multi-tenant service. Within one Gateway instance, authenticated control-plane access carries authority over the configured agents, sessions, and policies in that deployment.
