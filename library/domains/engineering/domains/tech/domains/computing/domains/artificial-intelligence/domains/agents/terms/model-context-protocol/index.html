Model Context Protocol

2026-03-29 An open protocol standardizing how language models connect to external tools and data sources — the agent-to-tool complement to A2A's agent-to-agent communication.

The Model Context Protocol (MCP) is an open protocol standardizing how agentic systems connect to external tools and data sources. Anthropic released MCP in November 2024. On December 9, 2025, it became a founding project of the Linux Foundation’s Agentic AI Foundation (AAIF), alongside AGENTS.md and Block’s goose. As of early 2026, the ecosystem includes 5,800+ servers, 300+ clients, and 97M+ monthly SDK downloads. OpenAI, Google, Microsoft, and all major IDEs have adopted it.

MCP follows a client-server architecture. The MCP server exposes capabilities — tools, resources, and prompts — through a standardized interface. The MCP client (embedded in the agent runtime) discovers and invokes those capabilities. The protocol uses JSON-RPC 2.0 for message exchange, with transport over stdio (for local processes) or HTTP with Server-Sent Events (for remote servers).

Core primitives

Tools. Functions the model can invoke. Each tool has a name, description, and input schema defined in JSON Schema. The model decides when to call a tool based on the conversation context; the client executes the call against the server and returns the result. This is the same tool use pattern that providers implement natively, but MCP makes tool definitions portable across providers and runtimes.

Resources. Data the model can read — files, database records, API responses, structured documents. Resources have URIs and MIME types. The client fetches resources on behalf of the model, injecting their content into the context window. Resources extend what the model can see; tools extend what the model can do.

Prompts. Reusable prompt templates that servers can expose for common tasks. The client can list available prompts and fill in their parameters.

Relationship to A2A

MCP and A2A address orthogonal concerns. MCP connects an agent to its tools — databases, file systems, APIs, code execution environments. A2A connects agents to other agents — systems with their own tools, models, and implementations. An agent uses MCP to act on its environment and A2A to coordinate with peers. PydanticAI integrates both natively.

Relationship to native tool use

Every major LLM provider (OpenAI, Anthropic, Google) implements tool use through proprietary API parameters. MCP sits above this layer: it defines how tools are described, discovered, and invoked regardless of which provider’s API the agent uses. A LangGraph workflow, a PydanticAI agent, and a Claude Code session can all connect to the same MCP server and use the same tools, even though they use different models and different runtimes.
