¶Control Architectures and Patterns

Edge-Cloud Hybrid:

  • Building automation: 5-15 min interval at edge; long-term optimization in cloud
  • Smart homes: Local LLM for voice; cloud for learning
  • Agriculture: Local GPIO control; cloud for model training
  • Power grid: SCADA edge; GridMind coordinates across agents

Hierarchical Decomposition:

  • Brain-LLM: High-level planning (goal interpretation)
  • Body-LLM: Motion planning (detailed waypoint generation)
  • Low-level controller: Actuator commands (motor speeds, GPIO pins)
  • Closed-loop feedback: Sensor readings replan if path blocked

Human-in-the-Loop:

  • GridMind: AI recommends; operator approves
  • Buildings: AI controls within constraints; humans override if needed
  • Smart homes: AI acts; humans can immediately counter-command
  • Agriculture: Pilot phase has human verification; production may be autonomous with fallback

¶Communication Protocols

Industrial stack: Modbus RTU (equipment) → Modbus TCP (networks) → OPC UA (semantic) → REST APIs (enterprise)

IoT-focused: MQTT (publish-subscribe, lightweight) primary for agricultural/smart home systems

Building-specific: BACnet, Modbus TCP, OPC UA

Power grid: SCADA legacy (DNP3, IEC 60870-5-104); OPC UA increasingly adopted

See full protocol hierarchy in “Protocols and Integration Patterns” text.

¶Safety Mechanisms

Input validation: Constrain observation space (grounding variables) Output validation: Whitelist action spaces, check against equipment registry Constraint enforcement: Physical ranges (temperature 65-85°F), rate limits, equipment capabilities Fallback safety: Graceful degradation to safe default on AI failure Audit logging: All decisions logged with reasoning + approval chain Monitoring: Continuous drift detection, alert if performance drops

¶Failure Modes

Data quality: Distribution shift, insufficient training examples Integration: Protocol changes, communication loss, stale data Security: Sensor spoofing, DDoS, malicious input injection Emergent: Multiple agents interact unexpectedly (storage charging simultaneously strains grid) RPA-specific: Brittleness to UI changes, unintended scale (bot approves 100x budget due to currency conversion)

See “Failure Mode Analysis” in Safety text for details.

¶Regulatory and Standards

ISO/IEC TS 22440: AI-specific functional safety standard (2024, not yet mandatory)

  • Risk classification by autonomy + criticality
  • Continuous data monitoring for model drift
  • Explainability requirements
  • Governance for retraining cycles

EU Machinery Regulation (takes effect 2027):

  • Learning phase constraints
  • Emergency stop, collision avoidance, geofencing
  • Fallback behavior on AI failure

NIST AI Risk Management Framework: Draft; expected to inform US regulatory approach

Timeline: 2027 likely inflection point (EU regulation takes effect); 2028+ expect domain-specific standards

¶Common Questions Answered in Research

Q: Can AI agents operate without human supervision? A: Not in critical systems. All production deployments observed use human-in-the-loop (AI recommends, human approves or overrides). See “Practical Safety Deployment Patterns.”

Q: What’s the difference between RPA and agentic AI? A: RPA is deterministic rule-based; agentic AI reasons adaptively. They’re complementary: agents route decisions, RPA executes well-defined tasks. See “RPA and Enterprise API Integration.”

Q: How do AI agents handle the real-time requirements of critical systems? A: Edge-cloud hybrid: fast responses at edge (local controllers), slower optimization in cloud. See “Edge-Cloud Hybrid” in Protocols text.

Q: What safety standards apply to AI-controlled physical systems? A: ISO/IEC TS 22440 (2024, not yet mandatory); EU Machinery Regulation (2027); mostly industry-led development still. See “Regulatory Framework Emergence” in Safety text.

Q: Why are RPA projects failing at 30-50% rates? A: Mostly human error: wrong process selected for automation, insufficient governance, poor adaptation to process changes. Not technology failure. See “RPA Failure Characteristics.”

Q: Which companies lead in AI physical systems control? A: Honeywell (buildings), Argonne Lab (power grid), Home Assistant (smart homes), Automation Anywhere/UiPath (RPA), DJI (agriculture drones). See “Organizations and Products by Domain” in Summary.
