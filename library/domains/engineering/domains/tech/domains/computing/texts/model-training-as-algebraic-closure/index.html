Model Training as Algebraic Closure

2026-04-04 The mathematical structure of LLM training maps onto algebraic closure operators: pre-training closes over a corpus, fine-tuning applies an additional nucleus, in-context learning temporarily extends the generator set, and the trained model is the fixed point.

A closure operator on a set is a function cl that is extensive (S is a subset of cl(S)), monotone (if S is a subset of T then cl(S) is a subset of cl(T)), and idempotent (cl(cl(S)) = cl(S)). The elements of cl(S) that were not in S are the consequences generated by closing over S. A fixed point of cl is any set F where cl(F) = F — closure adds nothing new.

This structure describes LLM training exactly.

Pre-training: closure over the corpus

The training corpus is a generator set G. Each document contributes token sequences that define the empirical distribution P(token | context). Pre-training applies a closure operator T to G: starting from random weights W_0, each gradient step moves the model toward a state where its predicted distribution matches the empirical one. The loss function is cross-entropy: L = -log(p_correct), and each update adjusts weights proportional to the gradient of this loss (Vaswani et al., 2017).

The process is extensive: the model’s representational capacity encompasses the training data. It is monotone: adding data to the corpus can only expand what the model captures. It is idempotent: once the model has converged to weights W* where the training loss is minimized, reapplying the same training process yields T(W*) = W*. The trained model IS the fixed point.

Convergence means the model has internalized not just individual sequences but the statistical regularities across the entire corpus — the closure of G under next-token prediction includes all the distributional consequences that follow from the generator set.

Fine-tuning: an additional nucleus

A nucleus (in the algebraic sense) is a closure operator applied to an already-closed object, producing a sub-fixed-point that satisfies additional constraints. Fine-tuning does exactly this.

LoRA (Hu et al., 2021) makes the structure visible: weight updates are confined to a low-rank subspace, decomposed as delta-W = BA where B and A have rank r much smaller than the model dimension. The pre-trained weights stay frozen. The nucleus acts within a constrained subspace, tightening the fixed point without rebuilding it. The result is a new fixed point W** satisfying both the original pre-training closure and the fine-tuning constraint.

RLHF (Ouyang et al., 2022) applies a preference-based nucleus. A reward model r encodes human judgments, and PPO optimization finds weights that maximize reward while a KL-divergence penalty prevents drifting too far from the pre-trained fixed point. The penalty is the algebraic constraint: the nucleus must produce a sub-fixed-point of the original closure, not an arbitrary new one.

DPO (Rafailov et al., 2023) compresses this into a single operator by expressing preference satisfaction directly in terms of the policy, eliminating the intermediate reward model. The composite closure — pre-training followed by preference alignment — reaches its fixed point in one pass.

In each case, fine-tuning does not reopen the original closure. It applies a second closure operator whose fixed points are a subset of the first’s.

In-context learning: temporary generators

In-context learning changes the model’s behavior without changing its weights. A prompt containing examples or instructions temporarily extends the generator set: instead of computing P(y | x) from the fixed weights alone, the model computes P(y | x, prompt). The weights W* remain the stable fixed point. The prompt is an ephemeral addition to G that shifts the output distribution for the duration of the context window.

This is not a new closure. It is the same closure operator applied to a temporarily enlarged generator set. When the context window ends, the extension vanishes and the model returns to its base fixed point. No weights changed; no new fixed point was established.

The correspondence

The CCE specification operationalizes this pattern for structured cells: its apply function is a closure operator, its layered expansion is the iterative construction of cl(G), and its fuel bounds ensure termination. The voidmath normalization process (Act F3.1 applied until Thing F5.3 is reached) is the same structure: generators produce histories, closure normalizes them, and the fixed point is the fully settled layer.

The claim is not metaphorical. Pre-training satisfies the three axioms of a closure operator. Fine-tuning satisfies the definition of a nucleus. In-context learning satisfies the definition of generator-set extension. The trained model satisfies the definition of a fixed point. The mathematics is the same mathematics.
