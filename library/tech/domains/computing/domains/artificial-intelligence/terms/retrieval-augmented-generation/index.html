Retrieval-Augmented Generation

2026-03-29 A pattern where a language model's generation is grounded by retrieving relevant documents from an external knowledge base and injecting them into the context window before generating a response.
<p>Retrieval-augmented generation (RAG) is a pattern where a <a href="../on-device-inference/terms/large-language-model.md" class="link-internal">large language model</a> retrieves relevant documents from an external knowledge base before generating a response. The retrieved documents are injected into the <a href="./context-window.md" class="link-internal">context window</a> alongside the user&rsquo;s query, grounding the model&rsquo;s generation in specific source material. Lewis et al. introduced the term and architecture in &ldquo;Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks&rdquo; (NeurIPS 2020).</p> <p>The standard RAG pipeline has three stages:</p> <ol> <li> <p><strong>Indexing.</strong> Documents are split into chunks, each chunk is converted into a vector <a href="./embedding.md" class="link-internal">embedding</a>, and the embeddings are stored in a vector database (Pinecone, Weaviate, Chroma, pgvector, or similar).</p> </li> <li> <p><strong>Retrieval.</strong> When the user submits a query, the query is embedded using the same embedding model. The vector database returns the k most similar document chunks by cosine similarity (or another distance metric).</p> </li> <li> <p><strong>Generation.</strong> The retrieved chunks are inserted into the model&rsquo;s prompt — typically between the <a href="./system-prompt.md" class="link-internal">system prompt</a> and the user&rsquo;s message. The model generates its response with access to this retrieved context.</p> </li> </ol> <p>RAG addresses a specific limitation of language models: their knowledge is frozen at training time. A model trained through January 2025 knows nothing about events after January 2025. RAG provides access to current information, proprietary data, and domain-specific documents that the model was never trained on — without retraining or <a href="./fine-tuning.md" class="link-internal">fine-tuning</a>.</p> <p>RAG also reduces <a href="./hallucination.md" class="link-internal">hallucination</a>. A model generating from memory may produce plausible but false statements. A model generating with relevant source documents in its context window is more likely to reproduce information from those documents accurately. RAG does not eliminate hallucination — the model can still ignore or misinterpret the retrieved context — but it shifts the failure mode from fabrication to misreading.</p> <h3 id="rag-and-agentic-systems">RAG and agentic systems</h3> <p>In <a href="../domains/agents/terms/agentic-system.md" class="link-internal">agentic systems</a>, RAG is one strategy for managing the <a href="./context-window.md" class="link-internal">context window</a> constraint. An agent that accumulates tool results and reasoning traces over many steps eventually exceeds the window. RAG allows the agent to store intermediate results in a knowledge base and retrieve them when needed, trading window space for retrieval latency.</p> <p>RAG can also be implemented as a <a href="../domains/agents/terms/tool-use.md" class="link-internal">tool</a>: the agent calls a &ldquo;search&rdquo; tool that queries a vector database and returns relevant chunks. This is functionally equivalent to RAG but gives the model control over when and what to retrieve, making retrieval a decision the agent makes rather than a fixed pipeline step.</p> <h2 id="related-terms">Related terms</h2> <ul> <li><a href="./embedding.md" class="link-internal">Embedding</a> — the vector representations that enable semantic retrieval</li> <li><a href="./hallucination.md" class="link-internal">Hallucination</a> — the failure mode RAG reduces</li> <li><a href="./context-window.md" class="link-internal">Context window</a> — the budget RAG competes for</li> <li><a href="../domains/agents/terms/agent-memory.md" class="link-internal">Agent memory</a> — the broader problem RAG partially addresses</li> </ul>
