Embedding

2026-03-29 A fixed-length vector of floating-point numbers that represents a piece of text (or image, or audio) in a continuous space where semantic similarity corresponds to geometric proximity.
<p>An embedding is a fixed-length vector of floating-point numbers that represents a piece of text (or image, or audio) in a continuous space where semantic similarity corresponds to geometric proximity. Two sentences about the same topic produce vectors that are close together (high cosine similarity); two sentences about unrelated topics produce vectors that are far apart.</p> <p>Mikolov et al. introduced the foundational word embedding models (Word2Vec) in &ldquo;Efficient Estimation of Word Representations in Vector Space&rdquo; (2013), demonstrating that <a href="./neural-network.md" class="link-internal">neural networks</a> trained on word co-occurrence patterns produce vector spaces with algebraic structure — the famous example: vector(&ldquo;king&rdquo;) - vector(&ldquo;man&rdquo;) + vector(&ldquo;woman&rdquo;) ≈ vector(&ldquo;queen&rdquo;).</p> <p>Modern embedding models (OpenAI <code>text-embedding-3</code>, Cohere Embed, Google Gecko, open-source models like <code>bge</code> and <code>e5</code>) produce embeddings for entire sentences or documents, not just individual words. These models are <a href="./transformer.md" class="link-internal">transformers</a> trained specifically for the embedding task — they process a text input and output a single vector (typically 768–3072 dimensions) that captures the text&rsquo;s semantic content.</p> <h3 id="how-embeddings-work">How embeddings work</h3> <p>The embedding model processes input text through its <a href="./transformer.md" class="link-internal">transformer</a> layers and produces a vector from the final layer&rsquo;s output — typically by averaging <a href="./token.md" class="link-internal">token</a> representations (mean pooling) or using a dedicated [CLS] token. The model is trained so that texts with similar meaning produce similar vectors: training pairs texts that are semantically related (from paraphrases, search queries and their results, or question-answer pairs) and pushes their embeddings closer together while pushing unrelated pairs apart (contrastive learning).</p> <p>The resulting vector space has structure: similarity search (finding the k nearest vectors to a query vector) functions as semantic search. A query about &ldquo;how to fix a broken pipe&rdquo; retrieves documents about plumbing repair, even if those documents never use the exact phrase &ldquo;broken pipe.&rdquo;</p> <h3 id="role-in-agentic-systems">Role in agentic systems</h3> <p>Embeddings are the enabling technology for <a href="./retrieval-augmented-generation.md" class="link-internal">retrieval-augmented generation</a> and vector-based <a href="../domains/agents/terms/agent-memory.md" class="link-internal">agent memory</a>. The standard pipeline: chunk documents into passages, embed each passage, store the vectors in a vector database, and retrieve the most relevant passages for a given query by comparing query and passage embeddings.</p> <p>In <a href="../domains/agents/terms/agentic-system.md" class="link-internal">agentic systems</a>, embeddings provide the search layer that lets an agent find relevant information in large knowledge bases without scanning every document. The agent calls a search tool, the tool embeds the query, the vector database returns the nearest matches, and the agent reads the retrieved documents in its <a href="./context-window.md" class="link-internal">context window</a>.</p> <h2 id="related-terms">Related terms</h2> <ul> <li><a href="./retrieval-augmented-generation.md" class="link-internal">Retrieval-augmented generation</a> — the primary application of embeddings in agent architectures</li> <li><a href="../domains/agents/terms/agent-memory.md" class="link-internal">Agent memory</a> — vector-based memory relies on embeddings</li> <li><a href="./neural-network.md" class="link-internal">Neural network</a> — the model architecture that produces embeddings</li> <li><a href="./transformer.md" class="link-internal">Transformer</a> — the specific architecture used by modern embedding models</li> </ul>
