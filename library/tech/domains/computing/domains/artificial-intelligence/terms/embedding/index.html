Embedding

2026-03-29 A fixed-length vector of floating-point numbers that represents a piece of text (or image, or audio) in a continuous space where semantic similarity corresponds to geometric proximity.

An embedding is a fixed-length vector of floating-point numbers that represents a piece of text (or image, or audio) in a continuous space where semantic similarity corresponds to geometric proximity. Two sentences about the same topic produce vectors that are close together (high cosine similarity); two sentences about unrelated topics produce vectors that are far apart.

Mikolov et al. introduced the foundational word embedding models (Word2Vec) in “Efficient Estimation of Word Representations in Vector Space” (2013), demonstrating that neural networks trained on word co-occurrence patterns produce vector spaces with algebraic structure — the famous example: vector(“king”) - vector(“man”) + vector(“woman”) ≈ vector(“queen”).

Modern embedding models (OpenAI text-embedding-3, Cohere Embed, Google Gecko, open-source models like bge and e5) produce embeddings for entire sentences or documents, not just individual words. These models are transformers trained specifically for the embedding task — they process a text input and output a single vector (typically 768–3072 dimensions) that captures the text’s semantic content.

How embeddings work

The embedding model processes input text through its transformer layers and produces a vector from the final layer’s output — typically by averaging token representations (mean pooling) or using a dedicated [CLS] token. The model is trained so that texts with similar meaning produce similar vectors: training pairs texts that are semantically related (from paraphrases, search queries and their results, or question-answer pairs) and pushes their embeddings closer together while pushing unrelated pairs apart (contrastive learning).

The resulting vector space has structure: similarity search (finding the k nearest vectors to a query vector) functions as semantic search. A query about “how to fix a broken pipe” retrieves documents about plumbing repair, even if those documents never use the exact phrase “broken pipe.”

Role in agentic systems

Embeddings are the enabling technology for retrieval-augmented generation and vector-based agent memory. The standard pipeline: chunk documents into passages, embed each passage, store the vectors in a vector database, and retrieve the most relevant passages for a given query by comparing query and passage embeddings.

In agentic systems, embeddings provide the search layer that lets an agent find relevant information in large knowledge bases without scanning every document. The agent calls a search tool, the tool embeds the query, the vector database returns the nearest matches, and the agent reads the retrieved documents in its context window.

Related terms

  • Retrieval-augmented generation — the primary application of embeddings in agent architectures
  • Agent memory — vector-based memory relies on embeddings
  • Neural network — the model architecture that produces embeddings
  • Transformer — the specific architecture used by modern embedding models
