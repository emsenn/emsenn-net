Context Window

2026-03-29 The maximum number of tokens a language model can process in a single interaction — the hard constraint that shapes agent architecture, prompt design, and memory management.

The context window is the maximum number of tokens a large language model can process in a single interaction. Everything the model reads and generates — the system prompt, the user’s message, the conversation history, tool definitions, tool results, and the model’s own output — must fit within this fixed budget. Once the context window is full, earlier content must be dropped or compressed to make room for new information.

The constraint is architectural. The transformer’s self-attention mechanism computes pairwise attention scores between all tokens, requiring O(N²) compute and O(N) memory for a sequence of N tokens. Extending the context window requires proportionally more resources per forward pass. Research on efficient attention (sparse attention, sliding window, linear approximations) reduces the constant factor but does not eliminate the fundamental scaling relationship.

Context window sizes have grown rapidly:

Model Year Context window
GPT-3 2020 4,096 tokens
GPT-4 2023 8,192 / 32,768 tokens
Claude 3 2024 200,000 tokens
Gemini 1.5 Pro 2024 1,000,000 tokens
Claude Opus 4.6 2025 200,000 / 1,000,000 tokens

Even at one million tokens, the context window is finite and shapes the design of agentic systems in three ways:

  1. Memory management. An agent operating over many steps accumulates tool results and reasoning traces that eventually exceed the window. Agent orchestration frameworks must implement strategies for summarizing, compacting, or selectively retaining context — a problem analogous to memory management in operating systems.

  2. Prompt budgeting. A system prompt, tool definitions, conversation history, and instruction files (AGENTS.md, SOUL.md) all compete for the same token budget. Longer instructions leave less room for the task itself.

  3. Retrieval vs. inclusion. When relevant information exceeds the context window, the system must retrieve it on demand (RAG, tool calls to databases) rather than including it wholesale. The context window determines the boundary between what the model can hold “in mind” and what it must look up.

Context window size is a binding constraint on agent capability. A larger window allows more complex multi-step reasoning without information loss. A smaller window forces more aggressive summarization and more frequent tool calls to re-fetch information.

Related terms

  • Token — the unit in which context windows are measured
  • Transformer — the architecture whose attention mechanism creates the constraint
  • System prompt — a major consumer of context window budget
  • Agentic system — systems whose architecture is shaped by context limits
