Slash Commands and Skills

2026-03-30 User-defined and bundled commands in Claude Code invoked with /name — skills carry SKILL.md instructions loaded on demand rather than at session start.
Defines Slash Commands, Skills, SKILL.md
Requires claude-code

Slash commands are Claude Code’s mechanism for invoking task-specific instructions on demand. A user types /name at the prompt to load a command’s instructions into the conversation context. Claude Code ships with built-in commands (/help, /clear, /compact, /context, /review) that perform fixed operations, and supports user-defined skills backed by SKILL.md files.

A skill is a slash command with its own instruction document. Each SKILL.md has YAML frontmatter (name, description) and markdown body (the instructions Claude follows when the skill is invoked). The name field becomes the /slash-command. The description field serves double duty: it labels the command for the user and tells Claude when to load the skill automatically — if a user’s request matches the description, Claude may invoke the skill without being asked.

Skills can be placed at three locations: project skills (.claude/skills/<name>/SKILL.md) are version-controlled and shared with the team; user skills (~/.claude/skills/<name>/SKILL.md) are personal and available across all projects; plugin skills are installed via /plugin from published packages.

The key distinction from CLAUDE.md is temporal: CLAUDE.md loads on every session regardless of relevance, while skills load only when invoked or when Claude judges them relevant. This makes skills the primary mechanism for progressive disclosure — keeping root instructions minimal while allowing deep task-specific guidance to exist in the project without consuming context on every request.

Skills consume context when loaded. The budget scales dynamically at 2% of the context window, with a 16,000-character fallback. Skills exceeding the budget may be excluded; /context surfaces warnings. The limit is overridable via the SLASH_COMMAND_TOOL_CHAR_BUDGET environment variable.

As of late 2025, skills can be bundled as plugins — installable packages of slash commands, subagents, MCP servers, and hooks distributed and installed with /plugin.
