2026-03-18 The FARS encoding of a Specification: a markdown file with YAML frontmatter in the spec/ locale. Reference this spec when writing, locating, or validating a spec file in the Flatfile Agential Resource System.

flatfile-agential-resource-system-specification

What this is

This spec describes how a Specification is encoded and stored in the Flatfile Agential Resource System. In FARS, a Specification is a markdown Resource file living in the spec/ locale. It carries YAML frontmatter that identifies the concept being specified and declares the predicates that apply to it.

File layout

Simple specifications with no colocated files live at spec/{id}.md. A specification with colocated skills, runbooks, or scripts MUST live in a subdirectory:

spec/{id}/{id}.md
spec/{id}/skills/
spec/{id}/runbooks/

The directory name MUST equal the frontmatter id exactly.

Encodings

YAML frontmatter

Required keys:

KeyTypeValue
idkebab-case stringUnique within spec/. Filename MUST be {id}.md.
descriptionstring, ≤200 charsTwo sentences: what it defines, when to reference it.
date-createdYYYY-MM-DDDate the file was first written. Set once; never changed.
date-modifiedYYYY-MM-DDDate the file was last substantively changed.

A spec that fully defines a type MUST carry:

KeyTypeValue
definesCamelCase stringThe name of the type or concept this spec defines.

A spec that characterizes a type without fully defining it MUST carry:

KeyTypeValue
describesCamelCase stringA related type this spec elaborates without fully defining.

Optional keys:

KeyTypeValue
domaintype expressionInput type, for specs defining a transformation or skill.
codomaintype expressionOutput type, for specs defining a transformation or skill.

No other frontmatter keys are permitted. Unknown keys are a compliance violation. Every key used in frontmatter MUST have a corresponding spec file in spec/ — see predicate.
